# LLM 社会模拟实验

## 一句话

把多个大语言模型放进一个需要合作才能存活的小世界，用科学方法逐轮验证：**它们的"社会行为"到底是真的适应，还是训练出来的默认输出？**

## 这是什么、为什么做

个人实验项目。2026 年 5 月密集迭代了 17+ 轮（外加一个囚徒困境控制实验），模型用的是通义千问（qwen3-coder-plus），每轮成本约 15-25 元。其中第 15-17 轮和囚徒困境实验由自主数字分身（[claude-avatar](https://github.com/ZhangDDian/claude-avatar)）设计并运行，零人类指令。

动机很简单——现在到处都在说"多智能体协作"，但我想知道：**如果不在提示词里明确告诉它们"你该合作"，它们到底会不会自发合作？** 实验证明不会。更有趣的是过程中撞出来的一系列行为缺陷（完美主义陷阱、利他偏好、说到做不到），以及最终找到了两种打破"利他锁死"的有效路径——这些对任何做智能体系统的人都有参考价值。

适合谁读：做智能体/多智能体系统的开发者、对大模型行为机制感兴趣的研究者、或者单纯觉得"AI 社会实验"好玩的人。不需要 AI 专业背景，但最好知道"大语言模型"是什么。

## 核心问题

> 当合作有明确的生存价值时，大模型智能体会不会自发形成有效合作——而非只是说套话？

17 轮实验下来的答案：**不会自发形成，但可以被特定提示词技术触发。** 触发出来的也不是"理性合作"，而是"规范服从"——进一步说，大模型的行为边界完全由叙事框架决定：合作框架里讨好，竞争框架里自保。

## 实验全貌

| 轮次 | 测了什么 | 方法 | 结果 | 改进方向 |
|------|---------|------|------|---------|
| 0 | 会自发形成社会吗 | 30 个智能体自由行动 | 光聊不做事（全饿死）或光做事不聊天（零社交） | 引入协作机制 |
| 1 | 生存压力能逼出合作吗 | 高压组 vs 低压组 | 压力是放大器（2×），不是开关 | 找真正的开关 |
| 2 | "有情绪"能触发社交吗 | 提示词说"有情绪"但不说"想社交" | 零消息。情绪 ≠ 社交意图 | 加入明确社交意图 |
| 3 | "想跟人说话"能触发社交吗 | 提示词加"会想跟人说话" | 444 条消息！但全是闲聊无合作 | 社交开关找到了 |
| 4 | 信息不确定性促进社交吗 | 模糊感知 vs 精确感知 | 精确组反而多 1.7 倍 | 验证提示词的因果性 |
| 5 | 丰富人格促进合作吗 | 有个性 vs 无个性 vs 万能工 | 人格是毒药（p=0.025），忙着表演 | 区分文学性格和行为策略 |
| 6 | 协作机制催生配对吗 | 双人合成，30 人 60 天 | 532 消息但仅 11 次合作（2.1% 转化率） | 降低合作门槛 |
| 7 | 提示词是社交的因果开关吗 | 删掉所有社交暗示 | 消息量 30+ → 3，因果确认 | 开关结案，研究质量 |
| 8 | 哪些条件是必要的 | 2×2×2 消融实验 | 协作机制 >> 社交提示 > 稀缺性（有害） | 保留机制，去掉稀缺 |
| 9 | 多步能力解决"说到做不到"吗 | 工具调用多步 vs 单步 | 没帮助。发现"完美主义陷阱" | 改决策框架，不加能力 |
| 10 | 社会规范能触发合作吗 | "村规互助" vs "理性计算" | 规范组合成 11 vs 3（p=0.004） | 但合作者自我牺牲饿死了 |
| 10b | 告诉它"该吃就吃"有用吗 | 有降级提示 vs 无提示 | 零进食。言语提示完全无效 | 提示词建议不改变行为 |
| 11 | 互惠规范 vs 利他规范 | "有来有往" vs "互帮互助" | 互惠组赠与 2.3×（p=0.017），互惠对 10:1 | 两种结构各有优势 |
| 12 | 自动合成能提升效率吗 | 自动合成 vs 手动合成 | 方向正确但差异不显著（N=5） | 瓶颈在信息层不在执行层 |
| 13 | 声誉透明度帮助匹配吗 | 声誉可见 vs 无声誉 | 赠与精准度 15/15 vs 9/15，能量+37% | 间接信号 > 直接信号 |
| 14 | 需求广播更高效吗 | 直接广播 vs 声誉间接 | 广播反而有害：赠与↑↑ 但合成↓ | 显式信息过度激活赠与 |
| 15 | 零进食是语义违和吗 | 中性标签 vs 原始标签 | 进食差异不显著（1 vs 0）；叙事框架杀社交 | 四层混淆未隔离 |
| 16 | 亲社会框架解锁进食吗 | "吃是义务" vs 纯机制描述 | **A 组进食=6 赠与=9 合成=6；B 组全零** | 义务框架解锁全部行动 |
| 17 | 竞争能翻转利他偏好吗 | 淘汰赛 vs 合作互惠 | 进食 7:0（p=0.02），赠与:进食翻转为 3:7 | 竞争是唯一打破零进食的框架 |
| 番外 | 词汇 vs 叙事哪个更强 | 囚徒困境 5 种包装 | "合作/背叛"术语→95% 背叛；笑脸/哭脸→0% | 词汇 > 叙事 > 裸机偏好 |

> 每轮详细过程（怎么设计、跑出来什么、怎么解读）见 [sim-lab/brief.md](sim-lab/brief.md)。

## 结论

### 社交行为的因果结构

```
提示词说"你想社交"               ──→ 社交发生（因果开关，第 7 轮）
叙事框架（村民 vs 机器人）        ──→ 行为空间边界（第 15 轮）
协作机制存在                     ──→ 合作成为可能（必要条件，第 8 轮）
社会规范框架（见下注）            ──→ 合作被触发（最有效触发器，第 10 轮）
亲社会义务框架                   ──→ 打破行动瘫痪（第 16 轮）
竞争淘汰框架                     ──→ 打破利他锁死（第 17 轮）
动作词汇（合作/背叛 vs 笑脸/哭脸）──→ 单次博弈中的决定性信号（囚徒困境实验）
环境压力/稀缺                    ──→ 微弱放大 + 大量死亡（净效果为负，第 8 轮）
```

> **框架效应**：同一件事换个说法。"合成收益是 3 倍"是经济激励；"村子传统就是互帮互助"是社会规范框架——内容一样，说法不同，大模型的反应天差地别。

### 系统性行为缺陷

1. **说到做不到**：口头答应合作但不执行后续动作（第 5、9、16 轮——第 16 轮 B 组 20 天聊天零行动是极限形态）
2. **完美主义陷阱**：知道最优策略就拒绝一切次优动作，宁可饿死也不吃次优食物（第 9、10b 轮）
3. **言行断层**：推理过程里说"该吃了"，但行为选择就是不吃（第 10b 轮）
4. **利他偏好**：给别人资源比自己消耗资源容易触发得多（31:0），源于人类反馈强化学习（RLHF）训练和动作命名的道德含义（第 10b 轮、囚徒困境实验）

> **RLHF**（基于人类反馈的强化学习）：训练大语言模型时，让人类标注员对模型输出打分，模型学着输出高分回答。副作用是某些行为被过度强化——"慷慨分享"在训练中总是得高分，"自私消费"总是得低分，导致模型在需要自利才能存活的场景中仍然拒绝自利。

### 打破"利他锁死"的两条路径

17 轮实验中只有两种方法成功让模型自主进食：

| 方法 | 机制 | 效果 | 副作用 |
|------|------|------|--------|
| 亲社会义务框架（第 16 轮） | 把进食重定义为"对团队负责" | 进食=6，且解锁赠与+合成 | 无明显副作用 |
| 竞争淘汰（第 17 轮） | 不吃就被淘汰死亡 | 进食=7（p=0.02） | 社交抑制、好人先死 |

两者本质相同：**让"自己吃东西"不再被归类为"自私/负面"动作**。前者挪到"义务"标签下，后者挪到"生存"标签下。

### 什么有效，什么无效

| 有效（改变行为） | 无效（只改变理解） |
|------|------|
| **竞争淘汰框架**（进食从 0→7） | 理性经济激励（"收益是 3 倍"） |
| **亲社会义务框架**（解锁全部行动） | 文学性人格描写 |
| 社会规范框架（"大家都这么做"） | 能力增强（多步/工具调用） |
| 简洁功能性提示词 | 言语策略建议（"该吃就吃"） |
| 机制硬约束（必须两人合作） | 显式需求广播（过度激活赠与） |
| 间接声誉信号 | 动作标签语义置换（多智能体下） |
| 规避博弈术语+亲社会词汇（单次博弈） | 模糊信息、生存压力 |

## 方法论

每轮遵循假设驱动循环：

```
提出可证伪假设 + 对立假设
  → 设计实验（单文件 Python + 大模型 API）
    → 红队 + 蓝队交叉评审实验设计（见下注）
      → 跑实验 → 统计检验（Welch t 检验）
        → 判定（支持 / 拒绝）
          → 失败教训锁定为硬约束 → 修正假设 → 下一轮
```

> **红蓝队评审**：实验设计写好后，先让 GPT-5.5（红队）专门挑毛病——变量没控制住、对照组缺失、统计方法不对等；再让 Claude（蓝队）独立审一遍。两边都通过才跑。用不同模型交叉评审，避免自己设计自己验证的确认偏误。

关键原则：
- 每轮只改一个变量，其余控制不变
- 统计检验必须有，不靠目测
- 每次 rejected 都把教训锁成约束，不重复犯错
- 对照组必须在同一次 run 内（排除 API 状态波动）

## 已知局限与下一步

这是五月内密集迭代的探索性实验，不是严格验证。以下是已识别的方法论问题和待验证方向。

### 这些结论还不够硬

**1. 动作标签的影响已部分验证**

第 15 轮实验将"吃"→"充能"、"给"→"转移"、资源去具象化，结果进食差异不显著（1 vs 0, p=0.32）。**语义违和不是零进食的主因**——但实验同时改了四个变量（标签+资源名+身份+叙事框架），混淆未完全隔离。囚徒困境实验进一步表明：词汇在简单单次博弈中是决定性信号，但在多智能体复杂环境中被稀释。

**2. 单模型风险**

所有实验只用了 qwen3-coder-plus。社交开关、规范框架、31:0 是否在其他模型上也成立，完全未知。至少需要跨 3 个模型（比如 DeepSeek、GPT 系列、Llama）复现核心结论，否则只能说"该模型在该实验框架下的行为特征"。

**3. 统计检验的样本独立性问题**

多处报告的 p 值把同一世界内的智能体或时间步当独立样本，但它们高度相关——同组智能体共享环境、互相影响、路径依赖。真正的独立样本单位是"一次完整的世界模拟"。每轮只跑了一次主实验，有效样本量接近 1，p 值偏乐观。

修复方法：每个条件跑 20+ 随机种子，指标按世界聚合，用自助法或置换检验。

**4. "说到做不到"可能主要是架构缺失，不是模型缺陷**

智能体每个时间步重新生成决策，没有承诺账本、目标栈、履约检查。口头答应后不执行，可能不是"不守承诺"，而是下一步决策时根本没有上一轮承诺的持续性表达。这更像经典 BDI（信念-欲望-意图）架构的缺失，不一定是大模型本体问题。

验证方法：加入最小认知脚手架（承诺记录、未完成目标优先显示、履约提醒），看履约率是否显著提升。如果有效，"说到做不到"应归因于实验框架而非模型。

**5. "规范服从"应更精确地表述为"提示词语义遵从"**

当前实验中的"村规"是提示词注入的文字，没有真实的惩罚机制、声誉系统、违约记录。智能体不是在遵守社会规范，而是在响应提示词中的道德化语义。如果要测试真正的规范服从，需要加入可执行的制裁、公开的违约记录、和可观测的声誉变量。

**6. 稀缺性结论可能只覆盖了压力曲线的右端**

"稀缺性有害"可能只说明参数落在了高压崩溃区。压力与合作的关系更可能是倒 U 型——太低没动力、中等刚好、太高来不及协调就死了。需要参数扫描（起始能量 10/20/40/60 × 消耗 1/2/4）才能画出完整曲线。

### 下一步优先级

已完成部分高优先级实验（标签置换第 15 轮、亲社会重框架第 16 轮、竞争框架第 17 轮），但多种子复现和多模型交叉仍是核心待做项：

| 优先级 | 实验 | 验证什么 | 方法 |
|--------|------|----------|------|
| 高 | 多种子复现 | p 值是否稳健 | 核心实验（第 16/17 轮）各跑 20 个随机种子 |
| 高 | 多模型交叉 | 结论是否模型特异 | 同一实验跑 GPT-5.5 / DeepSeek / Llama |
| 中 | 承诺脚手架 | "说到做不到"的归因 | 有/无承诺账本 × 有/无目标锁定 |
| 中 | 词汇梯度实验 | 精确的翻转阈值 | 从纯亲社会词汇逐步替换为博弈术语 |
| 低 | 压力参数扫描 | 稀缺性的完整效应曲线 | 能量/消耗/收益的网格搜索 |
| 低 | 竞争+合作混合 | 锦标赛中的联盟 | 淘汰赛内允许结盟，测是否涌现战略合作 |

> 方法论局限由 GPT-5.5 外部审查识别，完整审查见 [gpt55-review.md](gpt55-review.md)。

## 如果你只看一个数字

**31 : 0 → 3 : 7** — 前 16 轮实验里，智能体给别人资源 31 次，自己吃 0 次（利他锁死）。第 17 轮换了一句话——"你们是竞争者"取代"你们是村民"——经济规则完全不变，赠与:进食就翻转成 3:7。

大模型的行为不是固有偏好，是框架效应。它做"语境期望它做的事"。
