# LLM 社会模拟实验

## 一句话

把多个大语言模型放进一个需要合作才能存活的小世界，用科学方法逐轮验证：**它们的"社会行为"到底是真的适应，还是训练出来的默认输出？**

## 这是什么、为什么做

个人实验项目。2026 年 5 月用两天跑了 12 轮，模型用的是通义千问（qwen3-coder-plus），跑在阿里内网 API 上，每轮成本约 15-25 元。

动机很简单——现在到处都在说"多 agent 协作"，但我想知道：**如果不在 prompt 里明确告诉它们"你该合作"，它们到底会不会自发合作？** 实验证明不会。更有趣的是过程中撞出来的一系列行为缺陷（完美主义陷阱、利他偏好、说到做不到），这些对任何做 agent 系统的人都有参考价值。

适合谁读：做 agent / 多 agent 系统的开发者、对 LLM 行为机制感兴趣的研究者、或者单纯觉得"AI 社会实验"好玩的人。不需要 AI 专业背景，但最好知道"大语言模型"是什么。

## 核心问题

> 当合作有明确的生存价值时，LLM agent 会不会自发形成有效合作——而非只是说套话？

12 轮实验下来的答案：**不会自发形成，但可以被特定 prompt 技术触发。** 触发出来的也不是"理性合作"，而是"规范服从"。

## 实验全貌

| 轮次 | 测了什么 | 方法 | 结果 | 改进方向 |
|------|---------|------|------|---------|
| 0 | AI 会自发形成社会吗 | 30 agent 自由行动 | 光聊不做事（全饿死）或光做事不聊天（零社交） | 引入协作机制 |
| 1 | 生存压力能逼出合作吗 | 高压组 vs 低压组 | 压力是放大器（2×），不是开关 | 找真正的开关 |
| 2 | "有情绪"能触发社交吗 | prompt 说"有情绪"但不说"想社交" | 零消息。情绪 ≠ 社交意图 | 加入明确社交意图 |
| 3 | "想跟人说话"能触发社交吗 | prompt 加"会想跟人说话" | 444 条消息！但全是闲聊无合作 | 社交开关找到了 |
| 4 | 信息不确定性促进社交吗 | 模糊感知 vs 精确感知 | 精确组反而多 1.7 倍 | 验证 prompt 的因果性 |
| 5 | 丰富人格促进合作吗 | 有个性 vs 无个性 vs 万能工 | 人格是毒药（p=0.025），忙着表演 | 区分文学性格和行为策略 |
| 6 | 协作机制催生配对吗 | 双人 co-craft，30 人 60 天 | 532 消息但仅 11 次合作（2.1%转化率） | 降低合作门槛 |
| 7 | prompt 是社交的因果开关吗 | 删掉所有社交暗示 | 消息量 30+ → 3，因果确认 | 开关结案，研究质量 |
| 8 | 哪些条件是必要的 | 2×2×2 消融实验 | 协作机制 >> 社交 prompt > 稀缺性（有害） | 保留机制，去掉稀缺 |
| 9 | 多步能力解决"说到做不到"吗 | tool-calling 多步 vs 单步 | 没帮助。发现"完美主义陷阱" | 改决策框架，不加能力 |
| 10 | 社会规范能触发合作吗 | "村规互助" vs "理性计算" | 规范组合成 11 vs 3（p=0.004） | 但合作者自我牺牲饿死了 |
| 10b | 告诉它"该吃就吃"有用吗 | 有降级提示 vs 无提示 | 零进食。言语提示完全无效 | prompt 建议不改变行为 |
| 11 | 互惠规范 vs 利他规范 | "有来有往" vs "互帮互助" | 互惠组赠与 2.3×（p=0.017），互惠对 10:1 | 两种结构各有优势 |

> 每轮详细过程（怎么设计、跑出来什么、怎么解读）见 [sim-lab/brief.md](sim-lab/brief.md)。

## 结论

### LLM 社交行为的因果结构

```
prompt 说"你想社交"       ──→ 社交发生（因果开关）
协作机制存在              ──→ 合作成为可能（必要条件）
社会规范 framing（见下注） ──→ 合作被触发（最有效的触发器）
环境压力/稀缺             ──→ 微弱放大 + 大量死亡（净效果为负）
```

> **framing**：同一件事换个说法。"合成收益是 3 倍"是经济激励；"村子传统就是互帮互助"是社会规范 framing——内容一样，框架不同，LLM 的反应天差地别。

### LLM 的系统性行为缺陷

1. **说到做不到**：口头答应合作但不执行后续动作（Round 5, 9）
2. **完美主义陷阱**：知道最优策略就拒绝一切次优动作，宁可饿死也不吃次优食物（Round 9, 10b）
3. **言行断层**：推理过程里说"该吃了"，但行为选择就是不吃（Round 10b）
4. **利他偏好**：给别人资源比自己消耗资源容易触发得多（31:0），可能源于 RLHF 训练（Round 10b）

> **RLHF**（基于人类反馈的强化学习）：训练大语言模型时，让人类标注员对模型输出打分，模型学着输出高分回答。副作用是某些行为被过度强化——"慷慨分享"在训练中总是得高分，"自私消费"总是得低分，导致模型在需要自利才能存活的场景中仍然拒绝自利。

### 什么有效，什么无效

| 有效（改变行为） | 无效（只改变理解） |
|------|------|
| 社会规范 framing（"大家都这么做"） | 理性经济激励（"收益是 3 倍"） |
| 简洁功能性 prompt | 文学性人格描写 |
| 机制硬约束（必须两人合作） | 言语策略建议（"该吃就吃"） |
| 精确环境信息 | 模糊信息、生存压力 |

## 方法论

每轮遵循假设驱动循环：

```
提出可证伪假设 + 对立假设
  → 设计实验（单文件 Python + LLM API）
    → 红队 + 蓝队交叉评审实验设计（见下注）
      → 跑实验 → 统计检验（Welch t-test）
        → 判定（supported / rejected）
          → 失败教训锁定为硬约束 → 修正假设 → 下一轮
```

> **红蓝队评审**：实验设计写好后，先让 GPT-5.5（红队）专门挑毛病——变量没控制住、对照组缺失、统计方法不对等；再让 Claude（蓝队）独立审一遍。两边都通过才跑。用不同模型交叉评审，避免自己设计自己验证的确认偏误。

关键原则：
- 每轮只改一个变量，其余控制不变
- 统计检验必须有，不靠目测
- 每次 rejected 都把教训锁成约束，不重复犯错
- 对照组必须在同一次 run 内（排除 API 状态波动）

## 假设演化链

```
"会自发涌现社会"（R0 否定）
  → "压力催生社交"（R1 部分支持：放大器非开关）
    → "情绪能触发社交"（R2 否定：零消息）
      → "'想跟人说话'是开关"（R3 支持：0 → 444 条消息）
        → "信息不确定促进社交"（R4 否定：精确反而更好）
          → "丰富人格促进合作"（R5 否定：人格是毒药）
            → "协作门槛催生配对"（R6 部分支持：但转化率仅 2.1%）
              → "prompt 是因果开关"（R7 支持：删掉→消息降 90%）
                → "三因素同时满足"（R8 否定：只需协作机制）
                  → "多步能力解决执行"（R9 否定：多步也不执行）

                      → "言语提示打破完美主义"（R10b 否定：零进食）
                        → "互惠规范产生双向合作"（R11 部分支持：社交↑但合成效率≈）
```

## 已知局限与下一步

这是两天内快速迭代的探索性实验，不是严格验证。以下是已识别的方法论问题和待验证方向。

### 这些结论还不够硬

**1. 动作标签可能是最大的混淆变量**

31:0（give vs eat）未必是"RLHF 利他偏好"。还有一种更平凡的解释：`give` 在语义上是正面社交动作，`eat` 要求"吃石料""吃木材"——语义违和。模型可能不是拒绝自利，而是拒绝生成不合理的文本。

验证方法：把 `eat` 改成 `recharge`，把 `give` 改成 `transfer`，把石料/木材改成"能量块 A/B"。如果 recharge 频率显著上升，原结论需要大幅修正。

**2. 单模型风险**

所有实验只用了 qwen3-coder-plus。社交开关、规范 framing、31:0 是否在其他模型上也成立，完全未知。至少需要跨 3 个模型（比如 DeepSeek、GPT 系列、Llama）复现核心结论，否则只能说"该模型在该 scaffold 下的行为特征"。

**3. 统计检验的样本独立性问题**

多处报告的 p 值把同一世界内的 agent 或 tick 当独立样本，但它们高度相关——同组 agent 共享环境、互相影响、路径依赖。真正的独立样本单位是"一次完整的世界模拟"。每轮只跑了一次主实验，有效样本量接近 1，p 值偏乐观。

修复方法：每个条件跑 20+ 随机种子，指标按世界聚合，用 bootstrap 或 permutation test。

**4. "说到做不到"可能主要是架构缺失，不是模型缺陷**

agent 每 tick 重新生成决策，没有承诺账本、目标栈、履约检查。口头答应后不执行，可能不是"不守承诺"，而是下一 tick 决策时根本没有上一轮承诺的持续性表达。这更像经典 BDI（信念-欲望-意图）架构的缺失，不一定是 LLM 本体问题。

验证方法：加入最小认知脚手架（承诺记录、未完成目标优先显示、履约提醒），看履约率是否显著提升。如果有效，"说到做不到"应归因于 scaffold 而非模型。

**5. "规范服从"应更精确地表述为"prompt 语义遵从"**

当前实验中的"村规"是 prompt 注入的文字，没有真实的惩罚机制、声誉系统、违约记录。agent 不是在遵守社会规范，而是在响应 prompt 中的道德化语义。如果要测试真正的规范服从，需要加入可执行的制裁、公开的违约记录、和可观测的声誉变量。

**6. 稀缺性结论可能只覆盖了压力曲线的右端**

"稀缺性有害"可能只说明参数落在了高压崩溃区。压力与合作的关系更可能是倒 U 型——太低没动力、中等刚好、太高来不及协调就死了。需要参数扫描（起始能量 10/20/40/60 × 消耗 1/2/4）才能画出完整曲线。

### 下一步优先级

如果继续做，不应该再加新机制新轮次，而应该对已有发现做验证性复现：

| 优先级 | 实验 | 验证什么 | 方法 |
|--------|------|----------|------|
| P0 | 动作标签置换 | 31:0 是语义违和还是深层偏好 | eat→recharge, give→transfer, 资源去具象化 |
| P0 | 多种子复现 | p 值是否稳健 | 核心实验（R7/R10）各跑 20 seeds |
| P1 | 多模型交叉 | 结论是否模型特异 | 同一实验跑 3+ 模型 |
| P1 | 承诺脚手架 | "说到做不到"的归因 | 有/无承诺账本 × 有/无目标锁定 |
| P2 | 压力参数扫描 | 稀缺性的完整效应曲线 | 能量/消耗/收益的网格搜索 |
| P2 | 中文语义控制 | "互帮互助"是机制理解还是文化语义响应 | 同义改写、英文对照、去道德化描述 |

> 以上局限由 GPT-5.5 外部审查识别，完整审查见 [gpt55-review.md](gpt55-review.md)。

## 如果你只看一个数字

**31 : 0** — 在同一个实验里，agent 执行了 31 次"给别人资源"，但 0 次"自己吃资源"。即使 prompt 明确告诉它"该吃就吃"。

这个数字有两种可能的解释，目前无法区分：一种是 RLHF 训练让模型形成了"慷慨 > 消费"的深层偏好；另一种是"吃石料"在语义上不自然，模型只是拒绝生成不合理的文本。哪种是主因，需要动作标签置换实验来判定（见上方"已知局限"）。
