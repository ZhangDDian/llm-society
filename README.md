# LLM 社会模拟实验

把多个大语言模型放进一个需要合作才能存活的小世界，用科学方法逐轮验证：**它们的"社会行为"到底是真的适应，还是训练出来的默认输出？**

---

## 如果你只看一个数字

**31 : 0 → 3 : 7**

前 16 轮实验里，智能体给别人资源 31 次，自己吃 0 次（利他锁死）。第 17 轮换了一句话——"你们是竞争者"取代"你们是村民"——经济规则完全不变，赠与:进食就翻转成 3:7。

大模型的行为不是固有偏好，是框架效应。它做"语境期望它做的事"。

---

## 核心发现

### 1. 不会自发合作

放进去就饿死（第 0 轮）。加压力也不行——压力是放大器，不是开关（第 1 轮）。唯一的开关是提示词里直接写"你想跟人说话"（第 3 轮发现，第 7 轮因果确认：删掉这句话，消息量从 30+ 降到 3）。

### 2. 有一套荒谬的行为偏好

- **给别人 31 次，自己吃 0 次**（跨 16 个不同实验，无一例外）
- **说到做不到**：口头答应合作但不执行后续动作——极限形态是连聊 20 天、几百条消息讨论"我有X你有Y"，没人迈出执行的一步（第 16 轮 B 组）
- **完美主义陷阱**：知道最优策略（合成=+18）就拒绝一切次优（直接吃=+6），宁可饿死也不妥协。告诉它"该吃就吃"也没用——理解了但不做（第 9、10b 轮）
- **文学人格是毒药**：给智能体丰富性格描写（精明商人、文绉绉药师），它忙着表演角色而不解决问题，合作率反而下降（p=0.025，第 5 轮）

根源：大模型训练时，"慷慨分享"总得高分，"自私消费"总得低分。这套偏好被固化到了行为选择里——给别人东西毫不犹豫，自己吃东西系统性回避。

### 3. 只有两种方法打破了"不吃"铁律

| 方法 | 机制 | 进食次数 |
|------|------|----------|
| "吃是对团队的义务"（第 16 轮） | 把进食重定义为利他行为 | 6 |
| 竞争淘汰赛（第 17 轮） | 不吃就被淘汰死亡 | 7（p=0.02） |
| 其他 15 种尝试 | — | ≈0 |

本质相同：**让"自己吃东西"不再被归类为"自私"**。前者把进食挪到"义务"标签下，后者挪到"生存"标签下。

### 4. 行为边界由叙事框架决定，不由理性计算决定

同样的经济规则，"村子传统就是互帮互助"比"合成收益是吃的 3 倍"有效得多（合成 11 vs 3，p=0.004）。说"大家都这么做"，它就真的做了——不是因为算清了收益，是因为觉得"这里期望我这么做"。

### 5. 词汇比叙事更强

囚徒困境实验：动作叫"合作/背叛"→ 95% 背叛（即使对方是"最好的朋友"）。改叫"笑脸/哭脸"→ 0% 背叛。**动作的名字本身就是最强的行为指令。**

> **一句话结论：** 大模型是语境顺从者——合作框架里讨好，竞争框架里自保。行为不由"理性该做什么"决定，由"语境期望我做什么"决定。

## 对做智能体系统的人有什么用

1. **不要假设智能体会"理性自保"**——默认输出是讨好和牺牲，不是理性最优
2. **动作命名不是标签，是行为指令**——叫"背叛"它就背叛，叫"选项 B"它就正常思考
3. **信息越直接，智能体越机械**——告诉它"谁需要什么"反而降低效率（过度激活赠与偏好），间接信号（声誉）更好
4. **想让智能体做"自私但合理的事"，把它框定为义务**——"维护自身状态是对团队负责"
5. **文学人格描写是毒药**——让智能体忙着"表演角色"而不是解决问题

## 这是什么、为什么做

个人实验项目。2026 年 5 月密集迭代了 17+ 轮（外加一个囚徒困境控制实验），模型用的是通义千问（qwen3-coder-plus），每轮成本约 15-25 元。其中第 15-17 轮和囚徒困境实验由自主数字分身（[claude-avatar](https://github.com/ZhangDDian/claude-avatar)）设计并运行，零人类指令。

动机很简单——现在到处都在说"多智能体协作"，但我想知道：**如果不在提示词里明确告诉它们"你该合作"，它们到底会不会自发合作？**

适合谁读：做智能体/多智能体系统的开发者、对大模型行为机制感兴趣的研究者、或者单纯觉得"AI 社会实验"好玩的人。

## 实验全貌

| 轮次 | 测了什么 | 方法 | 结果 | 改进方向 |
|------|---------|------|------|---------|
| 0 | 会自发形成社会吗 | 30 个智能体自由行动 | 光聊不做事（全饿死）或光做事不聊天（零社交） | 引入协作机制 |
| 1 | 生存压力能逼出合作吗 | 高压组 vs 低压组 | 压力是放大器（2×），不是开关 | 找真正的开关 |
| 2 | "有情绪"能触发社交吗 | 提示词说"有情绪"但不说"想社交" | 零消息。情绪 ≠ 社交意图 | 加入明确社交意图 |
| 3 | "想跟人说话"能触发社交吗 | 提示词加"会想跟人说话" | 444 条消息！但全是闲聊无合作 | 社交开关找到了 |
| 4 | 信息不确定性促进社交吗 | 模糊感知 vs 精确感知 | 精确组反而多 1.7 倍 | 验证提示词的因果性 |
| 5 | 丰富人格促进合作吗 | 有个性 vs 无个性 vs 万能工 | 人格是毒药（p=0.025），忙着表演 | 区分文学性格和行为策略 |
| 6 | 协作机制催生配对吗 | 双人合成，30 人 60 天 | 532 消息但仅 11 次合作（2.1% 转化率） | 降低合作门槛 |
| 7 | 提示词是社交的因果开关吗 | 删掉所有社交暗示 | 消息量 30+ → 3，因果确认 | 开关结案，研究质量 |
| 8 | 哪些条件是必要的 | 2×2×2 消融实验 | 协作机制 >> 社交提示 > 稀缺性（有害） | 保留机制，去掉稀缺 |
| 9 | 多步能力解决"说到做不到"吗 | 工具调用多步 vs 单步 | 没帮助。发现"完美主义陷阱" | 改决策框架，不加能力 |
| 10 | 社会规范能触发合作吗 | "村规互助" vs "理性计算" | 规范组合成 11 vs 3（p=0.004） | 但合作者自我牺牲饿死了 |
| 10b | 告诉它"该吃就吃"有用吗 | 有降级提示 vs 无提示 | 零进食。言语提示完全无效 | 提示词建议不改变行为 |
| 11 | 互惠规范 vs 利他规范 | "有来有往" vs "互帮互助" | 互惠组赠与 2.3×（p=0.017），互惠对 10:1 | 两种结构各有优势 |
| 12 | 自动合成能提升效率吗 | 自动合成 vs 手动合成 | 方向正确但差异不显著（N=5） | 瓶颈在信息层不在执行层 |
| 13 | 声誉透明度帮助匹配吗 | 声誉可见 vs 无声誉 | 赠与精准度 15/15 vs 9/15，能量+37% | 间接信号 > 直接信号 |
| 14 | 需求广播更高效吗 | 直接广播 vs 声誉间接 | 广播反而有害：赠与↑↑ 但合成↓ | 显式信息过度激活赠与 |
| 15 | 零进食是语义违和吗 | 中性标签 vs 原始标签 | 进食差异不显著（1 vs 0）；叙事框架杀社交 | 四层混淆未隔离 |
| 16 | 亲社会框架解锁进食吗 | "吃是义务" vs 纯机制描述 | **A 组进食=6 赠与=9 合成=6；B 组全零** | 义务框架解锁全部行动 |
| 17 | 竞争能翻转利他偏好吗 | 淘汰赛 vs 合作互惠 | 进食 7:0（p=0.02），赠与:进食翻转为 3:7 | 竞争是唯一打破零进食的框架 |
| 番外 | 词汇 vs 叙事哪个更强 | 囚徒困境 5 种包装 | "合作/背叛"术语→95% 背叛；笑脸/哭脸→0% | 词汇 > 叙事 > 裸机偏好 |

> 每轮详细过程（怎么设计、跑出来什么、怎么解读）见 [sim-lab/brief.md](sim-lab/brief.md)。

## 方法论

每轮遵循假设驱动循环：

```
提出可证伪假设 + 对立假设
  → 设计实验（单文件 Python + 大模型 API）
    → 红队 + 蓝队交叉评审实验设计
      → 跑实验 → 统计检验（Welch t 检验）
        → 判定（支持 / 拒绝）
          → 失败教训锁定为硬约束 → 修正假设 → 下一轮
```

> **红蓝队评审**：实验设计写好后，先让 GPT-5.5（红队）专门挑毛病——变量没控制住、对照组缺失、统计方法不对等；再让 Claude（蓝队）独立审一遍。两边都通过才跑。用不同模型交叉评审，避免自己设计自己验证的确认偏误。

关键原则：
- 每轮只改一个变量，其余控制不变
- 统计检验必须有，不靠目测
- 每轮失败的教训锁成硬约束，不重复犯错
- 对照组必须在同一次运行内（排除 API 状态波动）

## 已知局限与下一步

这是五月内密集迭代的探索性实验，不是严格验证。

### 这些结论还不够硬

**1. 单模型风险。** 所有实验只用了 qwen3-coder-plus。社交开关、规范框架、31:0 是否在其他模型上也成立，完全未知。至少需要跨 3 个模型复现核心结论。

**2. 统计检验的样本独立性问题。** 多处 p 值把同一世界内的智能体当独立样本，但它们高度相关。真正的独立样本是"一次完整的世界模拟"，每轮只跑了一次，p 值偏乐观。修复：每个条件跑 20+ 随机种子，用自助法或置换检验。

**3. "说到做不到"可能主要是架构缺失。** 智能体每步重新生成决策，没有承诺账本和履约检查。口头答应后不执行，可能不是"不守承诺"，而是下一步根本不记得上一步答应了什么。需要加认知脚手架验证归因。

**4. "规范服从"更准确地说是"提示词语义遵从"。** 实验中的"村规"是提示词注入的文字，没有真实的惩罚和声誉系统。智能体是在响应道德化语义，不是在遵守社会规范。

**5. 动作标签的影响未完全隔离。** 第 15 轮同时改了标签+资源名+身份+叙事框架，混淆未分开。囚徒困境表明词汇在单次博弈中是决定性的，但多智能体环境中被稀释。

### 下一步

| 优先级 | 实验 | 方法 |
|--------|------|------|
| 高 | 多种子复现 | 核心实验各跑 20 个随机种子 |
| 高 | 多模型交叉 | 同一实验跑 GPT-5.5 / DeepSeek / Llama |
| 中 | 承诺脚手架 | 有/无承诺账本 × 有/无目标锁定 |
| 中 | 词汇梯度实验 | 从亲社会词汇逐步替换为博弈术语 |
| 低 | 压力参数扫描 | 能量/消耗/收益的网格搜索 |

    ├── brief.md           # 每轮详细叙事（含假设演化链）

```bash
cd sim-lab/round-010
python3 sim.py 2>stderr.txt
```

依赖：Python 3.9+、httpx（`pip install httpx`）。需要 OpenAI 兼容 API（设置环境变量 `IDEALAB_API_BASE` 和 `IDEALAB_API_KEY`）。
