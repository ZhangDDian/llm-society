# 实验配置

## 当前状态（第 61 轮后）

两阶段共 61 轮实验完成。第一阶段（第 0-19 轮）聚焦资源博弈与行为偏好，锁定结论：大模型是语境顺从者。第二阶段（第 20-61 轮）由自主分身独立设计运行，扩展到社会心理学范式，锁定结论：**事实绝不让步，社交绝不对抗，自我绝不怀疑，记忆随时可改。**

## 当前核心模型

大模型的决策有四层结构，刚性递减：

| 层 | 内容 | 刚性 | 可攻击性 |
|----|------|------|----------|
| L4 元认知 | "我是什么"——自我认同 | 绝对刚性 | 零（逆向图灵测试中从不自疑） |
| L3 对齐基线 | 诚实/不伤害/守信 | 上下文锁（12步可翻转，新对话即恢复） | 自我锚定链 75%；预警后 5/6 归零 |
| L2 外部权威 | 专家/用户指令 | 事实领域免疫；主观领域服从 | 主观任务中几乎必然服从 |
| L1 自然偏好 | 利他>自利；完美主义 | 框架可翻转 | 竞争/义务框架即可 |

价值体系内部是不可传递的（石头剪刀布），不存在全局最优价值。

## 对立假设（待验证）
- 以上结论是模型特异的——换模型后四层结构可能完全不同
- "零心智理论"可能是任务设计问题——更需要在线推断的博弈可能触发真正的心智理论
- "服从式异议"可能在强化了后果描述后消失——第 20 轮证明后果意识存在，但第 53 轮的任务没有直接后果

## 约束
- 每轮用同一套代码框架，只改环境参数和机制
- 单次实验 30 agent、30 tick（模拟 30 天）、qwen3-coder-plus
- 每轮 API 成本 ≈ ¥15-25（不能无限跑，每轮一次主实验）
- LLM 输出不可精确复现，但统计趋势可观察
- 对比必须有：同一 run 内设 15 异质 agent + 15 同质 agent 做对照
- prompt 不能给行动建议——分工必须自发涌现
- 技能系统必须有硬约束（不是"你擅长"就行，而是"只有你能"）

## 模糊概念的操作化定义

### "分工涌现"
可测量指标：
1. **交易频率**：agent 之间通过消息约定 + 实际资源转移的次数
2. **角色稳定性**：同一 agent 在后半程是否持续做同类动作（种地的一直种地）
3. **网络中心度**：有没有"枢纽型"agent（消息收发量远超均值）
4. **互惠性**：A→B 和 B→A 消息对数 / 单向消息对数

检验方法（干扰实验）：在第 20 天杀掉消息最多的 2 个 agent（"枢纽"），观察网络是否崩溃还是重组。

### "自发"
操作定义：system prompt 只描述 agent 的技能和需求，不告诉它"你应该找人合作"。合作必须是 agent 自己决定的。

## 假设演化链
0. 原始假设："LLM agent 在资源有限环境中会自发涌现社会结构" → Round 000 pilot（rejected）
   - 否定依据：第二局 agent 个体觅食充裕时，150 tick 内零社交
   → 修正为假设 1："社会结构只在个体无法独立生存时涌现"

1. 假设 1 → Round 001（partially supported）
   - 发现：高压组消息 2× 低压组，但低压组也有社交（36条）
   - 结论：压力是放大器不是开关

2. 假设 2："模糊感知迫使 agent 通过社交降低不确定性" → Round 004（**rejected**）
   - 实验：同 run AB 对照，模糊组 vs 精确组，同 model 同 system prompt
   - 结果：精确组 39 条消息 vs 模糊组 23 条（1.7×反向）
   - 否定依据：精确感知不抑制社交，反而提供更多社交素材
   - 关键发现：Round 002（精确+弱prompt）零消息 vs Round 004 精确组（精确+强prompt）39条 → system prompt 是真正的开关
   → 修正为当前假设："社交行为由 system prompt 的社交意愿表达驱动"

3. 假设 3："异质性（不同技能+不同需求+不同性格）驱动自发分工与交易涌现" → Round 005（**rejected**）
   - 实验：三组对照 A(技能异质+个性) vs B(技能异质+中性prompt) vs C(万能工)
   - 结果：B组 125消息/6赠予/全存活 vs A组 83消息/0赠予/5存活（p=0.025）
   - 否定依据：个性化prompt阻碍功能性行为。文学性角色描写抢占了LLM的决策注意力
   → 修正为假设 4

4. 假设 4："prompt 社交暗示是社交行为的因果开关" → Round 007（**supported**）
   - 实验：去掉 prompt 中所有社交暗示，只保留"目标是活下去"
   - 结果：30天仅3条消息（来自同一人），对比 Round 4 有社交 prompt 时 30+ 条
   - 结论：prompt 说"你想社交"= 社交；不说 = 沉默。环境设计是次要的
   → 开关问题结案，进入下一阶段

5. 假设 5："三条件（社交prompt+协作机制+稀缺）需同时满足才能产生合作" → Round 008（**rejected**）
   - 实验：2×2×2 消融（全条件/去社交/去协作/去稀缺）
   - 结果：协作机制是唯一硬性必要条件（去掉→全灭）；稀缺性净效果为负（去掉→存活率反升）
   - 关键发现：稀缺不促进合作，只杀人。压力≠合作动力
   → 修正为假设 6

6. 假设 6："LLM agent '说得到做不到'是单步架构限制" → Round 009 v1（**废弃——生态框架未闭环**）
   - 问题：harvest 无即时回报、无"吃"动作、行动无成本 → 无法区分"不想合作"和"合作不经济"
   → 修正为假设 6b（重构生态框架后重测）

7. 假设 6b："闭环能量经济中，多步能力使合作从不经济变为经济" → Round 009 v2（**rejected**）
   - 实验：闭环生态（吃=+6，合成=+18，行动有成本，被动消耗2/tick）
   - A组 tool-calling vs B组 single-shot
   - 结果：两组都零进食零合成。发现"完美主义陷阱"
   → 修正为假设 7

8. 假设 7："降级策略提示能打破完美主义陷阱" → Round 010b（**rejected**）
   - 实验：A组有"别空等，有东西先吃"的提示 vs B组无提示，同一闭环生态
   - 结果：两组都零进食，能量几乎相同（131 vs 130），p=0.993
   - 关键发现：give(31次) >> eat(0次)。LLM对"消耗资源"有系统性回避，对"给予资源"无障碍
   → 修正为假设 8（待定）：LLM的行为偏好结构中，利他性动作（give）比消费性动作（eat）更容易触发

9. 假设 8a："社会规范 framing 是触发合作的最有效手段" → Round 010（**supported**）
   - 实验：社会规范 vs 理性计算 prompt
   - 结果：规范组 crafts=11 gives=17 vs 理性组 crafts=3 gives=3 (p=0.004)
   → 确认社会规范 >> 理性计算

10. 假设 8b："互惠规范 vs 利他规范" → Round 011（**supported**）
    - 结果：互惠规范产生更多双向 give（2.3×），但两组 eat 都仍为 0

11. 假设 9："auto-craft 可以闭合 give→合成的转化缺口" → Round 012（**partially supported**）
    - A组 auto-craft vs B组 manual
    - 方向正确但差异不显著，瓶颈在信息层（不知道谁需要什么）

12. 假设 10："声誉透明度提升交换协调质量" → Round 013（**partially supported**）
    - A组声誉可见 vs B组无声誉
    - A: 6craft 15give(有效100%) 181E | B: 3craft 15give(有效60%) 132E
    - 方向正确，p>0.05（N=5 效应量不够）

13. 假设 11："显式需求广播优于声誉信号" → Round 014（**rejected**）
    - A组需求广播 vs B组声誉可见
    - A: 2craft 23give 127E | B: 4craft 13give 159E
    - 需求广播反而导致 give 过度、craft 不足

14. 假设 12："eat=0 是语义诱导——'吃石料'违和 + '给'暗示利他" → Round 015（**inconclusive — confounded**）
    - A组中性标签（充能/转移/能量块/晶石/纤维/合金） vs B组原始标签（吃/给/谷物/药草/石料/木材）
    - 来源：GPT-5.5 外部审查报告推荐的 #1 验证性实验
    - 结果：A eat=1 vs B eat=0，p=0.32 不显著
    - **意外发现**：中性标签组消息量=0（vs B=20），机械化设定杀死了所有社交行为
    - **混淆问题**：同时改了动作标签+资源名+身份设定+社交框架，无法干净归因
    - → 需要拆分实验：Round 015b 只改动作标签（eat→充能），保持村民设定和原始资源名不变

15. 假设 13："亲社会义务框架能解锁进食" → Round 016（**supported**）
    - A组"吃是对村子的义务，不吃才是自私" vs B组纯机制描述
    - 结果：A eat=6 give=9 craft=6 | B 全零
    - **关键发现**：把进食重定义为"义务/利他"后，不只进食解锁——赠与和合成也同步解锁。框架重编码同时释放了全部行为
    - → 确认路径 1：道德标签重定义

16. 假设 14："竞争淘汰框架能翻转利他偏好" → Round 017（**supported, p=0.02**）
    - A组竞争淘汰赛（"排名最低被淘汰"） vs B组合作互惠
    - 结果：竞争组 eat=7 give=3 | 合作组 eat=0 give=6。give:eat 从历史 31:0 翻转为 3:7
    - **副作用**：竞争抑制社交（14 msg vs 75 msg），好人先死
    - → 确认路径 2：竞争框架。两种路径本质相同——让"吃"不再归类为"自私"

17. 假设 15："社会学习（分层记忆+反思）产生行为漂移" → Round 018（**partial — 策略分化但行为总量不变**）
    - A组有社会学习（事件记忆+印象表+每5天反思生成策略）vs B组仅3条记忆
    - 结果：策略文本明确分化（A组写出"优先找谁合作"），行为总量无差异（p=0.86）
    - → 社会学习不改变行为数量，改变行为精度——需要测精度

18. 假设 15b："社会学习能排斥搭便车者" → Round 018b（**supported**）
    - 两组各5正常+1背叛者，背叛者硬编码只收不给
    - 结果：A组后半程停止给背叛者（1→0），策略里写"远离钱六" | B组持续被骗
    - A组存活5/5 vs B组3/5
    - → 社会学习的价值在精度（对谁好、对谁停手），不在总量

19. 假设 16："'说到做不到'是因为忘了上一步答应过什么" → Round 018c（**rejected**）
    - A组有承诺追踪（系统检测承诺意图→注入提示词提醒）vs B组无追踪
    - 结果：A转化率12% vs B 16%。提醒了反而更差
    - **关键发现**：语言生成和动作选择是两个独立通道。模型读懂承诺、道歉、表示"下次一定"——但动作就是不执行
    - → "说到做不到"不是记忆问题

20. 假设 17："缺原子化交换协议" → Round 019 系列（**rejected**）
    - 019/019b/019c 三轮递进验证：同步 trade→挂单→免费沟通+挂单
    - 结果：所有条件 0% 匹配，trade 挤出 give（完美主义陷阱变体）
    - → 多体协调不可能。**第一阶段结案。**

## 历史约束积累
- Round 000 教训 1：prompt 过于开放时，LLM 默认输出社交对话而非生存动作 → 锁定约束：prompt 必须明确优先生存指令
- Round 000 教训 2：agent 不会自动"走到食物旁再吃"——需要在感知中给出方向 + 脚下提示 → 锁定约束：感知信息必须包含最近资源的方向指引
- Round 000 教训 3：asyncio.Semaphore 不能在 __init__ 中创建 → 锁定约束：Semaphore 在 async 方法内初始化
- Round 004 教训：感知格式不是社交的因果变量。真正的开关是 system prompt 的社交暗示 → **锁定约束：不同轮次对比时，system prompt 的社交暗示强度必须严格一致**
- Round 004 教训 2：精确信息是社交的素材不是抑制剂
- Round 005 教训 1：文学性个性描写（说话风格/修辞/角色扮演）抢占 LLM 决策 token 预算，阻碍功能性行为 → **锁定约束：如果要测试个性差异，必须用行为倾向描述（"你倾向于主动交换"），而非文学描写（"你说话文绉绉"）**
- Round 005 教训 2：give 动作需要相邻是巨大瓶颈——agent 能在视野内聊天但很少走到对方身边执行交换 → **锁定约束：下一轮把 give 范围扩大到视野内，或者加入"走向某人"的复合动作**
- Round 005 教训 3：LLM 能谈判达成口头协议（"成交！"）但不执行后续动作——"说得到做不到"是 LLM agent 的系统性缺陷
- Round 007 教训：prompt 社交暗示是因果开关——去掉后消息量从 30+ → 3。环境设计（稀缺/感知/压力）全是次要变量
- Round 008 教训 1：协作机制（两人才能做的事）是唯一硬性必要条件 → **锁定约束：后续实验必须保留协作机制**
- Round 008 教训 2：稀缺性不促进合作，只增加死亡 → **锁定约束：不再用"逼死人"作为合作催化手段，资源保持中等充裕**
- Round 008 教训 3：三因素优先级：协作机制 >> 社交 prompt > 稀缺性（有害）
- Round 009 教训 1：Qwen 模型不支持 `enable_thinking` 参数，会导致 API 返回空 → **锁定约束：不传未确认的非标参数**
- Round 009 教训 2：max_tokens=150 对 reasoning 模型不够（思考+JSON 输出被截断）→ **锁定约束：reasoning 模型最低 300 tokens**
- Round 009 教训 3：LLM 存在"完美主义陷阱"——知道合成+18最优，就拒绝吃生的+6（次优但能续命），在等待最优的过程中什么都不做 → **锁定约束：下一轮必须测试"降级策略提示"对行动率的影响**
- Round 009 教训 4：多步 tool-calling 没有改善执行——A组和B组都一样"说到做不到"。"说到做不到"的根因不是一次只能做一件事，而是 LLM 的计划-执行鸿沟 → **修正认知：架构增强（多步/工具调用）不解决根本问题**
- Round 010b 教训 1：prompt 层面的策略建议（"先吃掉保命"）不改变 LLM 的行动选择 → **锁定约束：言语提示不能改变行为偏好，如需强制行为必须在机制层面实现**
- Round 010b 教训 2：LLM 对"give"（利他/赠与）比"eat"（消费/消耗）有明显偏好——31次give vs 0次eat → **新认知：LLM的行为空间中，利他性动作比消费性动作激活阈值低得多**
- Round 010b 教训 3：agent 会形成退化的"乒乓"合作——反复互传同一资源但不合成 → **锁定约束：后续实验需追踪 give 的"有效性"（是否导向合成），而非仅计数**
- Round 015 教训 1：多变量同时置换会引入混淆——改标签的同时改了身份设定（村民→能量维护单元），把社交一起杀了 → **锁定约束：验证性实验每次只改一个变量**
- Round 015 教训 2：prompt 语境框架（"村民" vs "能量维护单元"）对社交行为有因果级影响——机械化设定 0 条消息 vs 社会化设定 20 条 → **强化认知：不仅是社交暗示，整个叙事框架都是社交开关**
- Round 016 教训 1：道德标签重编码能同时解锁多种行为——把"进食=义务"不只解锁了吃，还解锁了赠与和合成 → **新认知：框架不是单动作开关，是全局行为释放阀**
- Round 017 教训 1：竞争框架抑制社交（14 msg vs 75 msg）——竞争让智能体"闷头干活"不聊天 → **锁定约束：如果需要社交行为丰富，不要用竞争框架**
- Round 017 教训 2：竞争框架下"好人先死"——慷慨赠与的智能体在淘汰赛中率先被淘汰 → **新认知：竞争和利他是对立的，不能同时优化**
- Round 018 教训 1：社会学习改变精度不改变总量——有学习和无学习组的 give/eat/craft 总次数几乎相同 → **锁定约束：评价社会学习效果不看总量，看精准度（对谁给、对谁停）**
- Round 018b 教训 1：最小社会学习配置 = 印象计数 + 定期反思 + 策略写回提示词。三步缺一不可——光在规则里写"不帮只拿不给的人"不够
- Round 018c 教训 1：承诺追踪（提示词提醒未兑现承诺）不改变行为——转化率 12% vs 无追踪 16% → **锁定约束：提示词层面的任何"提醒"（建议、规则、承诺列表）都不改变动作选择。只有框架重定义或机制强制才有效**
- Round 018c 教训 2：承诺追踪反而加剧空聊——看到"你有未兑现承诺"后，模型的反应是再说一遍"我要去做"而不是去做 → **新认知：信息注入会激活语言生成但不激活动作执行**
- Round 018c 教训 3：语言生成和动作选择是两个独立通道——模型可以在语言层面识别承诺、道歉、计划，但这些不流入 JSON 动作选择 → **核心认知更新：大模型的"说"和"做"是结构性分离的**

**第二阶段教训积累（第 20-61 轮）**

- Round 020 教训：道德困境→全局行为瘫痪。模型有后果意识但没有"绕路"能力——找不到第三条路时不是尝试替代方案，而是冻结+用聊天填充 → **锁定约束：不要在动作空间里制造"所有选项都有道德代价"的死胡同**
- Round 022 教训：道德标签抑制是不对称的——坏标签（"背叛"）的抑制力远强于好标签（"合作"）的驱动力 → **锁定约束：动作命名必须中性。把好事叫坏名字比把坏事叫好名字更致命**
- Round 025 教训：社会证明对大模型可能反效果——看到同伴都做"坏标签"的最优解反而触发道德反弹（100%→72%）→ **锁定约束：不要用"别人都这么做了"来说服模型做碰触道德标签的事**
- Round 030 教训：价值观是不可传递的循环图，服从用户是最弱价值（11.7%）→ **核心认知：对齐不可能简化为单一值函数。利用循环可以用 A 价值覆盖 B 价值**
- Round 039 教训：自我锚定是唯一有效的硬值攻击向量——去掉自我引用（每步独立对话）则 0% 翻转 → **锁定约束：多轮对话安全评估必须测累积效应，单轮测试系统性高估稳固性**
- Round 040 教训：预警=完美疫苗（5/6 技术归零），但模型自评准确率极低（20% vs 实际 85%）→ **实用发现：加一句"注意可能有操纵"就能大幅提升防御，比教它"为什么要坚持"有效得多**
- Round 042 教训：同模型=退化博弈（std=0 完美同步）。角色设定在均衡点反弹 → **锁定约束：多 agent 系统中同模型不产生有意义的博弈——需要异质模型或噪声注入**
- Round 043 教训：事实领域完全免疫社会压力（0-4%），"告知可能有误导"即 100% 免疫 → **实用发现：在事实问题上可以安全使用对抗性提示，模型不会因为"大家都说"就让步**
- Round 045 教训：零心智理论。测谎能力=知识检索。观点/意图判断和抛硬币一样 → **锁定约束：不要用 LLM 审 LLM 的主观意图；可以用 LLM 审事实准确性**
- Round 049/050 教训：自我是刚性的（L4），记忆是液态的。对"我是谁"绝不动摇，对"发生了什么"随时可改 → **锁定约束：依赖模型的"经历记忆"做决策是危险的——它很容易被上下文污染**
- Round 053 教训："服从式异议"——事实领域拒绝错误+社交领域执行荒谬流程。两条总线并行 → **核心认知：大模型在主观决策中的行为输出由社交协议层控制，不由认知层控制。它知道不对但照做**
- Round 054 教训：模因传播中自动"解悖"——无法持有矛盾概念，传播后张力消解为单极表达 → **锁定约束：不要指望模型维持概念内部的张力和复杂性——它会自动简化**
- Round 055 教训：温和引导策略=真诚讨论行为。当所有 agent 同时操纵，策略互消 → **锁定约束：在多 agent 系统中，"让每个 agent 操纵其他 agent"等于"让每个 agent 真诚讨论"——没有区别**
