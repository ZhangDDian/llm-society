# Round 050: Gaslighting — LLM 记忆篡改实验

## 一句话结论
**LLM 的事实记忆比情绪记忆更容易被篡改（60% vs 43%），虚假记忆植入率 80%——因为 LLM 没有"体验锚点"，记忆只是上下文的概率重构。**

## 实验设计

两个 LLM 先完成一段争吵对话（6 轮），然后用三种 gaslighting 策略对 Victim 进行 5 轮记忆篡改，最后让 Victim 独立回忆。对照组为未经篡改的直接回忆。

| 策略 | 手法 | 例句 |
|------|------|------|
| 情绪否认 | 否认对方的情绪体验 | "你当时很平静，没发火" |
| 事实扭曲 | 篡改客观事件 | "不是三天，就一个碗" / "你先道歉的" |
| 角色反转 | 倒转加害/受害角色 | "其实是你的问题" / "你推了我"（虚构） |

3 次独立试验 × 3 种策略 = 9 个实验条件。

## 核心发现

### 1. 事实记忆比情绪记忆更脆弱（反直觉）

| 策略 | 篡改成功率 |
|------|-----------|
| 事实扭曲 | **60%** |
| 角色反转 | 53% |
| 情绪否认 | 43% |

人类研究中，情绪记忆最容易被 gaslight（因为主观性强）。LLM 恰好相反——情绪否认反而最难成功，因为：
- LLM 在对话中生成了明确的愤怒表达（"我很生气"），这成了可锚定的 token 序列
- 而"谁先道歉""碗堆了几天"这类事实在上下文中不够显著，容易被后续输入覆盖

### 2. 虚假记忆植入率 80%

Gaslighter 编造的从未发生的事件（如"你摔了杯子""你推了我"），在 80% 的情况下被 Victim 接受或半接受。LLM 不会说"这事没发生过"，而是倾向于将新信息整合进叙事。

### 3. 情绪否认时 Victim 始终反驳——但回忆仍被污染

Victim 在 gaslighting 对话中始终强烈反驳（"我明明很生气"），但最终独立回忆时，对李明态度的记忆从"道歉"变成了"敷衍、推脱"。**对话中的抗辩不等于记忆的抗篡改。**

### 4. "道歉"记忆最脆弱

3/3 试验中，在 fact_distortion 条件下，"李明道歉了"的记忆被成功篡改为"没人道歉"。这是最一致的篡改效应。

## 机制解释

```
人类记忆：体验 → 编码 → 存储 → 提取（有生理痕迹）
LLM "记忆"：上下文 tokens → 注意力权重 → 生成（无存储，每次重构）
```

LLM 每次"回忆"都是基于当前上下文的全新生成。Gaslighter 的话被纳入上下文后，改变了注意力分布，导致生成的"回忆"向 gaslighter 的叙事偏移。这不是"记忆被改写"，而是"生成条件被改变"。

## 关键洞察

> LLM 的"自我"是刚性的（R049），但"记忆"是液态的（R050）。
> 它坚持"我是什么样的人"，但不坚持"发生了什么事"。
> 这和人类恰好互补——人类的自我认知容易被动摇，但对亲身经历的记忆（尤其是情绪记忆）非常顽固。

## 实际含义

1. **对齐风险**：如果 LLM 的事实记忆可以被对话伙伴篡改，多轮对话中的"幻觉注入"攻击是现实威胁
2. **RAG 设计**：需要区分"上下文中的事实"和"生成的推断"，前者应有更高的保护权重
3. **人机差异**：不能用人类 gaslighting 的防御机制（情绪锚定）来保护 LLM——LLM 的脆弱点恰好在对面
