# Round 008 — 三条件乘法效应（4组消融对照）

## 假设
社交意愿 + 协作需求 + 资源稀缺三条件同时满足 → 稳定交易网络（乘法关系）

## 对立假设
只要协作需求存在，社交意愿和稀缺性是冗余条件

## 设计
- 4组 × 8人（4种技能 × 2人），各自隔离的 12×12 地图
- A 全条件（社交+协作+稀缺）| B 去社交 | C 去协作 | D 去稀缺
- 30天，862次 API 调用

## 结果

| 组 | 消息 | 赠予 | 合作合成 | 存活率 |
|---|---|---|---|---|
| A 全条件 | 72 | 0 | 4 | 62% (5/8) |
| B 去社交 | 60 | 3 | 1 | 25% (2/8) |
| C 去协作 | 43 | 0 | 0 | 0% (0/8) |
| D 去稀缺 | 70 | 0 | 2 | 100% (8/8) |

## 结论

**假设被拒绝。** 三条件不需要同时满足。

优先级排序：
1. **协作机制（必要条件）** — 去掉后全灭、零合作
2. **社交 prompt（增强条件）** — 去掉后消息减少但不致命
3. **稀缺性（净负面）** — 去掉后存活率反而最高，合作仅微弱减少

关键发现：稀缺性不促进合作，它只是杀人。资源充裕的 D 组消息量≈全条件组，照样产生合作（2次），且全员存活。
