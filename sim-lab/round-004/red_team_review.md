## 红队（GPT-5.5）

## 替代解释（≥2 条）：

1. **“模糊感知”不是增加了不确定性，而是增加了语言触发器**
   - 模糊组收到的是自然语言、主观体验式输入：  
     “好像北边有食物味道”“远远看到人影”“肚子有点饿”。
   - 这类输入天然更像一个角色扮演场景，会激活 LLM 的叙事/社交默认模式：  
     “我不确定，你看到什么了吗？”“我们一起找食物吧。”
   - 精确组收到坐标、数值、方向，更像算法题或导航任务，模型倾向于直接执行最优移动。
   - 所以消息多不一定来自“信息需求”，而可能来自**prompt 风格诱导了聊天行为**。

2. **“功能性消息”指标可能只是相关，不是因果**
   - 你定义的功能性消息是：  
     “消息中包含方向信息 + 接收者随后朝该方向移动”。
   - 但接收者可能本来就会往那个方向走，因为：
     - 它自己也感知到类似线索；
     - 食物梯度本来就在那个方向；
     - 模型常用启发式是“往北/往食物味道方向走”；
     - 四个方向里随机命中概率也不低。
   - 因此，“说了北，接收者往北走”不能证明接收者是因为消息而行动。
   - 这仍可被对立假设解释为：  
     **聊天是噪声，方向一致只是共同环境线索或随机基率造成的假功能性。**

3. **存活率不差不等于社交有价值，可能只是环境太宽松**
   - 如果食物密度足够高、能量消耗低、30 ticks 太短，那么即使 agent 随便走也能活下来。
   - 模糊组多聊天但不死，可能只是因为聊天没有显著惩罚，而不是聊天有帮助。
   - 对立假设可以解释为：  
     “它们确实在不知道干嘛时聊天，但这个世界太容易，所以聊天没有把它们害死。”

4. **模糊组的消息可能是“表达不确定性”，不是“交换信息”**
   - 例如：
     - “我好像闻到北边有食物。”
     - “我不确定我们该去哪。”
     - “你看到什么了吗？”
   - 这些话表面上有方向，但未必增加了接收者的信息量。
   - 如果接收者本来就知道北边有食物，或者消息内容只是重复环境提示，那么它不是信息交换，只是语言模型在复述 prompt。

5. **第 20 天重聚也可能是记忆/角色扮演伪结构**
   - 如果 agent 的上下文里保留了过往交互、名字、对话记录，那么打乱位置后它们“寻找老朋友”可能只是因为 LLM 记得名字，而不是因为之前社交产生了真实协作价值。
   - LLM 很擅长延续叙事：“我应该去找之前合作过的 X。”
   - 这仍然可能是文本连续性，而非社会关系的功能性。

## 设计偏见（参数/规则层面）：

1. **感知格式同时改变了“不确定性”和“语言风格”**
   - 模糊组是主观自然语言，精确组是坐标数字。
   - 这不是纯粹操纵不确定性，而是同时操纵了：
     - 叙事性；
     - 情绪性；
     - 社交暗示；
     - 可行动性；
     - 语言生成倾向。
   - 因此社交差异可能来自格式，而不是不确定性。

2. **精确组被设计成“无需沟通”**
   - 精确组直接知道食物坐标、距离、方向、能量值。
   - 在这种设定下，沟通本来就被结构性剥夺了价值。
   - 所以“精确组不聊天”不能强力证明“不确定性驱动社交”，因为实验已经让聊天对精确组没有用。

3. **模糊组被设计成“看起来应该聊天”**
   - “远远看到一个人影”“好像北边有味道”这类表达，会诱导模型问别人。
   - 如果 prompt 中出现其他 agent 的存在，LLM 很可能自动生成社交行为。
   - 这会偏向当前假设。

4. **功能性消息的判定标准太宽**
   - “消息含方向 + 随后朝该方向移动”没有排除：
     - 接收者本来就会这么走；
     - 共同感知导致同向移动；
     - 随机命中；
     - 食物分布导致的环境偏置。
   - 这个指标容易把相关性误判为因果性。

5. **通信如果没有成本，会夸大社交数量**
   - 如果说话不消耗能量、不占行动机会、不增加风险，那么 LLM 可以随便聊天。
   - 现实中功能性社交应该能在有成本时仍然带来收益。
   - 免费通信环境会让“默认聊天”看起来无害甚至有功能。

6. **左右半区可能不是完全等价**
   - 模糊组和精确组分别住在同一世界的左右两半。
   - 即便食物密度相同，具体分布、边界效应、初始位置、agent 距离都可能不同。
   - 如果模糊组刚好更容易遇到彼此，消息就会更多。

7. **样本量小且轮数短**
   - 20 个 agent，分组后每组只有 10 个。
   - 30 ticks 很短，第 20 tick 才打乱位置。
   - 少量随机差异可能被解释成结构性规律。

8. **已有线索可能污染解释框架**
   - Round 002 精确感知 19 天零消息，Round 003 模糊感知 4 天 9 条消息。
   - 这确实提示差异，但也可能说明：  
     精确 prompt 抑制语言输出，模糊 prompt 激活语言输出。
   - 不能直接推到“信息交换有功能”。

## 你建议的修改：

1. **把“感知不确定性”和“语言风格”解耦**
   - 增加四组，而不是两组：
     1. 精确信息 + 数字格式；
     2. 精确信息 + 自然语言格式；
     3. 模糊信息 + 数字/概率格式；
     4. 模糊信息 + 自然语言格式。
   - 如果只有自然语言模糊组聊天多，说明是语言风格效应。
   - 如果所有模糊组都聊天多，才更支持不确定性驱动。

2. **加入消息阻断/延迟/打乱实验**
   - 让模糊组 agent 继续生成消息，但：
     - 一组正常送达；
     - 一组消息被系统丢弃；
     - 一组消息随机发给别人；
     - 一组方向词被打乱。
   - 如果消息真的有功能，正常送达组应该比阻断/打乱组活得更好、找食物更快。
   - 如果差不多，就说明聊天没有信息价值。

3. **让接收者先提交“无消息情况下的行动计划”**
   - 每个 tick 中，先让 agent 根据自身感知决定一个 provisional action。
   - 然后再给它看别人消息，让它可以修改行动。
   - 功能性消息应定义为：  
     **消息导致接收者改变原本行动，并且改变后更接近食物或提高存活。**
   - 这样才能证明消息有因果作用。

4. **设置通信成本**
   - 例如：
     - 发送消息消耗能量；
     - 发送消息占用行动；
     - 每 tick 只能移动或说话二选一；
     - 说话暴露位置，引来竞争。
   - 如果模糊组仍然选择通信并提高生存率，才说明社交有功能。
   - 如果加成本后聊天消失或导致存活下降，就支持对立假设。

5. **加入“假消息/随机消息”对照**
   - 给一部分模糊组 agent 注入同等数量的随机聊天：
     - “我觉得北边有食物。”
     - “也许我们该往东。”
   - 如果真实消息组和随机消息组表现差不多，说明所谓社交没有信息价值。
   - 如果真实消息组显著更好，才支持功能性信息交换。

6. **检验信息是否是“新增信息”**
   - 只有当发送者知道接收者不知道的信息时，消息才算信息交换。
   - 例如设计局部视野：
     - A 能看到东边食物，B 看不到；
     - A 告诉 B；
     - B 因此向东移动并找到食物。
   - 如果双方本来都能感知同一方向，那不应算功能性消息。

7. **多随机种子、交叉左右半区**
   - 同一个设置跑很多 seed。
   - 模糊/精确组随机分配到左半区或右半区。
   - 或者同一组 agent 在不同 run 中交换条件。
   - 排除地形、食物分布、初始位置造成的差异。

8. **更严格的重聚测试**
   - 第 20 天打乱后，隐藏老朋友名字或清除显式对话记忆，只保留行为收益线索。
   - 或者制造一个新环境，让找老朋友没有叙事提示，只有协作收益。
   - 如果 agent 仍然优先重聚并表现更好，才说明关系有结构性功能。

## 一句话：如果实验结果支持假设，最可能的“假阳性”原因是什么？

最可能的假阳性是：**模糊感知的自然语言 prompt 触发了 LLM 的默认社交/角色扮演输出，而“功能性消息”指标又把共同环境线索导致的同向移动误判成了因果性信息交换。**
