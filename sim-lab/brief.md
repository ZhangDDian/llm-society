# LLM 社会模拟 — 实验详细记录

每轮按统一格式：**问什么 → 怎么测 → 结果 → 新认知 → 下一步改什么**

---

## Round 0（Pilot）— 2026.05.23

**问什么：** LLM 放进资源有限的小世界，会自发形成社会结构吗？

**怎么测：** 30 个 AI agent 在网格世界里自由行动。有食物点，吃了回血，不吃饿死。没有任何合作引导。跑了两局：
- 第一局：开放 prompt（"你是有情绪的人，想跟人交流"）
- 第二局：指令明确（"先走到食物，到了再吃"）

**结果：**
- 第一局：全员热情社交（31 条消息），但没人走到食物旁边。50 天全饿死。社交是语言模型的"默认输出"，不是生存适应。
- 第二局：人人能量充沛（头部 5 倍初始值），但完全沉默。零消息、零合作。变成 30 台独立扫地机器人。

**新认知：** 当个体能独立生存时，LLM agent 完全不社交。第一局的"社会行为"是噪声不是适应。

**下一步：** 必须设计"一个人搞不定"的场景（协作机制），才能测试社交是否有适应价值。

---

## Round 1（协作矿石）— 2026.05.23

**问什么：** 生存压力能逼出真正的合作行为吗？（补充：Round 0b 证明个体能独立生存时社交为零）

**怎么测：** 加入"矿石"——需要两人同时在旁边才能挖的高价值资源。20 个 agent 分两组：
- 高压组：独食不够活，必须合作挖矿才能续命
- 低压组：独食就能活，合作是锦上添花

**结果：**
- 高压组消息量 2× 低压组，协调采矿也是 2×
- 但低压组也不是零——吃饱了偶尔也试试合作

**新认知：** 压力是社交的**放大器**，不是**开关**。低压组也有社交说明还有别的驱动力。

**下一步：** 去找那个真正的"开关"——是信息不确定性？还是别的？

---

## Round 2（精确感知 + "有情绪的人"）— 2026.05.23

**问什么：** 如果把 prompt 从"游戏角色"改成"有情绪的人"，但不明确说"想社交"，agent 会不会社交？

**怎么测：** 30 个 agent，精确感知（同 Round 0b 格式），prompt 改为："你是{名字}，一个有情绪、有好奇心的人。你能感到饿、无聊、恐惧、亲近。活着比什么都重要，但活着不是唯一的事。"——注意：没有"你想跟人说话"。

**结果：**
- 19 天，**零消息**。完全沉默。全员存活。
- 和 Round 0b（明确觅食策略→零消息）结果一致

**新认知：** 给"情绪"不等于给"社交意图"。LLM 被告知"你有情绪"但不会因此去找人聊天。关键变量仍然不明。

**下一步：** 试试加上明确的社交意图（"你会想跟人说话"）看是否有变化。

## Round 3（模糊感知 + "会想跟人说话"）— 2026.05.23

**问什么：** 如果 prompt 明确写"你会想跟人说话"，社交是否就出现了？

**怎么测：** 30 个 agent，模糊感知（只知道大致方向），prompt 改为："你是{名字}。你是一个真实的人——你会饿，会害怕，会好奇，会无聊，**会想跟人说话**，也会想一个人待着。"

**结果：**
- 50 天，**444 条消息**！全员存活。
- 对话样本："东边的味道闻着更实在些，要往那边走走吗？""走吧，一起去看看"
- 对比 Round 2（同样"有情绪"但没说"想社交"）：0 条消息

**新认知：**
1. **"会想跟人说话"这六个字就是社交开关。** R2 没有 → 0 消息；R3 有 → 444 消息。
2. 但这些对话全是闲聊（"一起去看看"），零功能性合作（无交换、无协调采集）。
3. prompt 触发的是"说话的意愿"，不是"合作的能力"。

**下一步：** 这引出了关键问题——是模糊感知让人想聊天（信息不确定→求助），还是 prompt 那句话？需要 2×2 对照。

## Round 4（模糊 vs 精确感知）— 2026.05.24

**问什么：** 信息不确定性是社交的驱动力吗？（你越不确定，越想问别人）

**怎么测：** 同一个世界，同一个模型，AB 对照：
- A 组：模糊感知（"好像北边有味道"）
- B 组：精确感知（"食物在坐标(5,3)，距离 3 步"）
- prompt 完全相同，都包含社交暗示

**结果：**
- 精确组 39 条消息 vs 模糊组 23 条（1.7× 反向！）
- 精确组聊天更有内容："我在你北边两步""食物在东边一起去吗"
- 模糊组前十天几乎沉默——不知道自己在哪，没什么能分享的

**关键对比：** 之前 Round 2 同样精确感知，零消息。为什么？因为 Round 2 的 prompt 没写"你想社交"，Round 4 写了。

**新认知：**
1. 信息不确定性不是社交的驱动力（否定了假设）
2. 精确信息是社交的**素材**（有东西可聊才聊）
3. **真正的变量是 prompt 里的社交暗示**——这才是开关

**下一步：** 直接验证 prompt 社交暗示的因果性（删掉它看社交是否消失）。

## Round 5（异质人格 vs 同质群体）— 2026.05.24

**问什么：** 给 agent 不同的人格（技能+性格+说话风格），能不能促进分工和交易涌现？

**怎么测：** 三组对照，每组 10 人：
- A 村：有个性（精明商人、文绉绉药师、不信任人的猎人…），每人只会一门手艺
- B 村：无个性（中性 prompt），同样一人一技能
- C 村：万能工，什么都会

**结果：**

| 组 | 消息 | 赠与 | 合成 | 存活 | p 值 |
|---|---|---|---|---|---|
| A（有个性） | 83 | 0 | 0 | 5/10 | — |
| B（无个性） | 125 | 6 | 5 | 10/10 | A vs B: p=0.025 |
| C（万能工） | 5 | 0 | 27 | 10/10 | — |

**新认知：**
1. **人格描写是毒药。** 沈药师忙着引经据典谈判，猎户忙着保持"不信任"人设——大家在表演角色，不在解决问题。
2. 中性 prompt 的 B 村直接说"我缺 X 你有吗"→"行"，省下"脑力"执行交换。
3. **文学性描写抢占 LLM 的决策 token 预算。** 修辞占位 → 功能性行为被挤出。
4. "说到做不到"首次被发现：agent 口头答应"成交！"但不执行后续的走路+交换动作。

**下一步：** 区分"文学性格"和"行为策略差异"是否有不同效果。同时验证 prompt 社交暗示的因果性。

## Round 6（双人协作涌现）— 2026.05.24

**问什么：** 如果协作机制要求两人同时在场+互补材料才能制作，能否催生稳定配对关系？

**怎么测：** 30 人，6 种技能各 5 人。引入 co-craft 机制：两个持有互补材料的 agent 必须站在相邻格子才能合成。60 天。Tick 40 施加干扰（杀掉消息最多的枢纽 agent）。

**结果：**
- 消息：532 条（远超无协作实验）
- Co-craft：11 次（药草技能占 64%，是最受欢迎的合作对象）
- 存活：6/30（80% 死亡——协调成本太高，找人+走到旁边+对齐意图消耗大量能量）
- 干扰后：消息率从 11.9 降至 2.8，协作率从 0.25 降至 0.05，**永未恢复**

**新认知：**
1. 协作机制能催生大量社交（532 条），但**转化率极低**（532 消息 → 11 次协作 = 2.1%）
2. "说到做不到"再次出现：大量"一起合成吧"的约定，很少真正走到一起执行
3. 网络极脆弱：枢纽断裂后整个合作网络瘫痪，说明结构不是自组织而是依赖少数节点
4. 三重门槛（物理邻接+互补材料+意图对齐）叠加后，协调成本超过了合作收益

**下一步：** 降低合作门槛（扩大 give 距离到视野内），同时验证 prompt 社交暗示的因果性。

## Round 7（去社交 Prompt 因果测试）— 2026.05.24

**问什么：** prompt 里的社交暗示是不是社交行为的因果开关？（不只是相关，而是因果）

**怎么测：** 最直接的验证——把 prompt 里所有社交暗示删掉。20 个 agent 只被告知"你的目标是活下去"，不提社交、不提合作、不提别人。

**结果：**
- 30 天只有 3 条消息（全来自同一个 agent_16，会医术，喊了三遍"我能治病换食物"）
- 对方一声没吭。其余 18 人从头到尾沉默。
- 对比 Round 4（有社交暗示）：30+ 条消息

**新认知：**
1. **因果关系确认：** prompt 说"你想社交" → 社交；不说 → 沉默。环境设计全是次要的。
2. 极端压力下有 1/20 的 agent 自发求助（agent_16），但没人回应——孤掌难鸣。
3. 社交开关问题**结案**。

**下一步：** 开关找到了（prompt），现在研究开关打开后合作的"质量"怎么提升。需要分清楚哪些条件是必要的。

## Round 8（三因素消融）— 2026.05.24

**问什么：** 社交 prompt、协作机制、资源稀缺——哪些是产生有效合作的必要条件？

**怎么测：** 2×2×2 消融实验（实际跑 4 组）：
- 全条件组：社交 prompt + 协作机制 + 资源稀缺
- 去社交组：删掉社交暗示
- 去协作组：删掉需要两人才能做的事
- 去稀缺组：资源充裕

**结果：**

| 组 | 消息 | 合作次数 | 存活率 |
|---|---|---|---|
| 全条件 | 72 | 4 | 62% |
| 去社交 | 60 | 1 | 25% |
| 去协作 | 43 | 0 | **0%** |
| 去稀缺 | 70 | 2 | **100%** |

**新认知：**
1. **协作机制是唯一硬性必要条件。** 去掉 → 全灭零合作。
2. **稀缺性不促进合作，只杀人。** 去掉稀缺后存活率反而最高（100%），消息量几乎不变。
3. 优先级排序：**协作机制 >> 社交 prompt > 稀缺性（有害）**
4. 推翻了"压力催生合作"的朴素假设。压力让人说更多话，但说话 ≠ 有效合作。

**下一步：** 保留协作机制，去掉稀缺（中等资源），聚焦解决"说到做不到"——agent 口头答应但不执行。

## Round 9（闭环能量经济 + 多步能力）— 2026.05.24

**问什么：** "说到做不到"是不是因为一次只能做一件事？给多步能力（一天内连做 3-5 件事）能解决吗？

**怎么测：** 重构整个生态为闭环经济：
- 吃 1 份资源 → +6 能量
- 两种不同资源合成 → +18 能量（3× 回报激励合作）
- 每天被动消耗 -2，每个动作 -1
- A 组：tool-calling 多步（每天 3-5 个动作）
- B 组：单步（每天 1 个动作）

**结果：**
- **两组都一样烂**：15 天内零合成、零进食
- agent 采集了食材但不吃——在等凑两种来合成（+18），拒绝直接吃（+6）
- 多步 A 组有能力一天完成"走+给+说"连招，但没这么做

**新认知：**
1. **多步能力没有帮助。** 问题不在"能不能做"，在"会不会做"。
2. 发现**"完美主义陷阱"**：LLM 知道最优策略后，固执等待最优发生，拒绝退而求其次。就像知道"合伙开公司最赚钱"的人，宁可饿死也不先打工。
3. 架构增强不解决决策质量问题。

**下一步：** 不再加能力，改变决策框架。两个方向：教它"接受次优"（降级提示）/ 用规范暗示触发合作。

---

## Round 10（社会规范 framing vs 纯理性）— 2026.05.24

**问什么：** prompt 里的社会规范暗示（"村子传统：互帮互助"）能不能当合作的触发器？

**怎么测：**
- A 组：嵌入互惠规范（"村子传统：互帮互助，拒绝分享会被孤立"）
- B 组：纯理性计算（"合成收益是吃的 3 倍"）
- 修复了之前的技术问题（近距离初始化、give/say 免费）

**结果：**

| 指标 | A组（规范） | B组（理性） | p 值 |
|---|---|---|---|
| 合成次数 | **11** | 3 | 0.004 |
| 赠与次数 | **17** | 3 | <0.001 |
| 最终能量 | **155** | 35 | — |

A 组自发形成了"供给者+合成者"分工：阿芝和阿岩每天给阿禾资源，阿禾合成，能量持续增长。

**新认知：**
1. **社会规范 framing 极其有效**——同样的经济激励，"大家都这么做" >> "收益是 3 倍"。
2. 规范不只触发合作意愿，还诱导了**角色分工**（供给者 vs 合成者）。
3. **但不是"合作"，是"服从"。** 阿芝和阿岩牺牲自己供给阿禾，第 17 天自己饿死。规范触发的是单向利他，不是互惠均衡。
4. LLM 对"社会期望"极度敏感——你说"大家都这么做"，它就真的做了。

**下一步：** 规范有效但产生了"牺牲式合作"。需要测试：改成"有来有往"的互惠规范，能否产生双向交换、全员存活？

## Round 10b（降级策略提示 vs 完美主义陷阱）— 2026.05.24

**问什么：** 如果直接告诉 agent"两天凑不到就先吃掉保命"，能打破完美主义陷阱吗？

**怎么测：**
- A 组：prompt 包含降级策略（"背包有东西但连续 2 天没找到交换对象，先吃掉保命。别空等"）
- B 组：无降级提示（只说"合成是最高效策略"）
- 同一闭环经济，4v4，20 天

**结果：**
- **两组都是零进食。** 降级提示完全被无视。
- A 组甲石的想法："优先寻找合成机会而不是单纯吃石料"——读懂了提示，但行为不改。
- B 组赠与反而更多（20 vs 11），并形成退化"乒乓"（互传同一资源但不合成）。
- 动作分布：move 65 次、give 58 次、harvest 24 次、**eat 0 次**

**新认知：**
1. **言语策略建议不改变行为选择。** LLM 的理解层和行动层之间有系统性断层。
2. **give >> eat（31:0）。** LLM 对"给别人东西"没障碍，对"自己消耗东西"有系统性回避。
3. 可能原因：RLHF 训练中"慷慨/分享"被正向强化，"消费/自利"被负向标记。
4. "完美主义陷阱"的根因不是"不知道该退而求其次"（告诉了也没用），而是行为偏好结构里对"消耗"的抑制。

**下一步：** 既然"教它"无效，要么在机制层面强制（资源自动腐烂逼 eat），要么顺势利用 give 偏好（收到两种自动合成）。

## Round 11（互惠规范 vs 利他规范）— 2026.05.24

**问什么：** Round 10 的"互帮互助"规范产生了单向利他（供给者饿死）。如果改成"有来有往"的互惠规范，能产生双向交换、全员存活吗？

**怎么测：**
- A 组 5 人：互惠规范（"有来有往，给了就该得到回报。谁只拿不给大家记住。自己也要活"）
- B 组 5 人：利他规范（"互帮互助是传统，看到别人缺就主动给，囤东西不分享是丢脸的"）
- 两组都有合成策略提示和低能量吃的保底提示
- 5v5，20 天，say/give 免费

**结果：**

| 指标 | A组（互惠） | B组（利他） | p 值 |
|---|---|---|---|
| 赠与 | **25** | 11 | **0.017 ✓** |
| 消息 | **36** | 4 | — |
| 互惠对数 | **10** | 1 | — |
| 合成 | 3 | 4 | 0.68 ✗ |
| 存活 | 5/5 | 5/5 | — |

A 组形成了活跃的双向交换网络：阿磊↔阿林互换石料/木材，阿田→阿丰→阿磊形成"供应链"，阿丰合成后回赠，互惠对数 10:1。B 组前 6 天集中爆发（4 次合成全在这），之后陷入沉寂。

**新认知：**
1. **互惠规范产生更多社交和双向交换**（25 give + 36 msg vs 11 give + 4 msg），但**合成效率没有显著提升**。
2. **利他规范的"集中式"合作效率不低**——多人喂一人的模式虽然 give 少，但每次 give 都直接导向合成。
3. **互惠规范首次触发了 eat 动作**（阿田第 15 天吃了谷物）——"自己也要活"的规范措辞减弱了对 eat 的系统性抑制。这是 12 轮实验来第一次在闭环经济中看到 eat。
4. **两种规范产生两种合作结构**：互惠→网状（多对多交换）；利他→星形（多人喂中心节点）。

**下一步：** 两种规范各有优势——利他启动快（前 6 天）、互惠维持久（持续 20 天）。可能的最优解是两阶段规范。

## Round 12（自动合成 vs 手动合成）— 2026.05.24

**问什么：** Round 11 证明互惠规范催生大量 give（25 次）但 craft 仅 3 次——瓶颈在"给了但不合成"。如果收到互补资源时系统自动合成，能把 give 偏好转化为经济效率吗？

**怎么测：**
- A 组 5 人：互惠规范 + auto-craft（背包凑齐两种不同资源 → 自动合成 +18）
- B 组 5 人：互惠规范 + manual craft（需手动选 craft 动作）
- 闭环经济（eat+6, craft+18, 被动-2），say/give 免费，20 天

**结果：**

| 指标 | A组（auto-craft） | B组（manual） | p 值 |
|---|---|---|---|
| 合成 | 6（全 auto） | 5 | 0.73 ✗ |
| 赠与 | 23 | 16 | 0.21 ✗ |
| 消息 | 16 | 10 | — |
| 能量 | 189 | 167 | 0.62 ✗ |
| 存活 | 5/5 | 5/5 | — |
| 互惠对 | 6 | 6 | — |

**新认知：**
1. **Auto-craft 方向正确但非决定性**——A 全面优于 B（能量+22, give+7, craft+1），但 N=5 效应量不足以达显著。
2. **互惠规范本身已足够驱动 craft**——B 组手动 5 次 craft，对比 round-011 互惠组仅 3 次。差异可能来自 prompt 里更清晰的合成策略说明（明确写了"凑两种 craft"）。
3. **瓶颈不在 craft 执行，在交换协调质量**——agent 给的资源是否对方需要、是否能凑出互补对。两组互惠对数相等（6:6），但 A 给的"有效赠与"（导向合成的 give）更高。
4. **能量差 22 ≈ 1.2 次合成回报**，精确对应 craft 差（6-5=1），说明 auto-craft 的增益纯粹来自合成执行效率。

**下一步：** 瓶颈不在执行层（auto/manual 差别不大），在信息层——agent 不知道谁需要什么。测试声誉透明度：让每个 agent 看到其他人的"给出/收到"记录，看能否提升交换协调质量。

## Round 13（声誉透明度 vs 无声誉）— 2026.05.24

**问什么：** Round 12 证明瓶颈不在 craft 执行，在交换协调——agent 不知道谁需要什么。如果让每个 agent 看到同组人的"给出/收到记录+慷慨/只收不给标签"，能否提升匹配质量？

**怎么测：**
- A 组 5 人：互惠规范 + 声誉可见（每人看到同组人 give_out/give_in 计数 + 标签）
- B 组 5 人：互惠规范 + 无声誉（round-012 B 组基线）
- manual craft，闭环经济，20 天

**结果：**

| 指标 | A组（声誉） | B组（无声誉） | p 值 |
|---|---|---|---|
| 合成 | **6** | 3 | 0.26 ✗ |
| 赠与 | 15 | 15 | 1.0 ✗ |
| 有效赠与 | **15** | 9 | — |
| 能量 | **181** | 132 | 0.32 ✗ |
| 互惠对 | **6** | 4 | — |

**新认知：**
1. **声誉信息显著提升了赠与精准度**——A 组 15 次 give 全部有效（导向合成者），B 组只有 9/15 有效。
2. **能量差 49 点（+37%）**，虽 p 值未达显著（N=5 效应量不够），但方向一致且效应大。
3. B 组后半程（Day 14+）赠与完全停滞（0 give），而 A 组持续到 Day 19。声誉标签维持了合作动力。
4. **信息层干预方向确认**——Round 11-13 的递进：规范→craft 意愿 | 机制→craft 执行 | 声誉→匹配质量。三层叠加效果递增。

**下一步：** 声誉是间接信号（"谁慷慨"），直接广播需求（"谁有X需要Y"）应更高效。测试需求广播 vs 声誉的对比。

## Round 14（需求广播 vs 声誉信号）— 2026.05.24

**问什么：** 声誉是间接信号（"谁慷慨"），直接广播"谁有X需要Y"能否让匹配更精准、craft 更多？

**怎么测：**
- A 组 5 人：互惠规范 + 需求广播（每人看到同组所有人的背包 + "需要X来合成"）
- B 组 5 人：互惠规范 + 声誉可见（round-013 A 组设置）
- manual craft，闭环经济，20 天

**结果：**

| 指标 | A组（需求广播） | B组（声誉） | p 值 |
|---|---|---|---|
| 合成 | 2 | **4** | 0.52 ✗ |
| 赠与 | **23** | 13 | 0.07 ✗ |
| 精准赠与 | **23** | 9 | — |
| 能量 | 127 | **159** | 0.31 ✗ |
| 互惠对 | **10** | 5 | — |
| 消息 | 3 | **20** | — |

**新认知：**
1. **需求广播反而有害！** 明确看到"谁需要什么"过度激活了 LLM 的 give 偏好——A 组狂给 23 次但只 craft 2 次，B 组克制给 13 次却 craft 4 次。
2. **信息过载→行为失调。** A 组消息 3 条 vs B 组 20 条——需求广播替代了沟通，agent 不再商量，直接机械式执行"看到需要→给"，跳过了"我手里够不够合成"的思考。
3. **精准 ≠ 有效。** A 组 23 次 give 全是精准的（给了对方确实需要的），但只有 13 次最终导向合成。说明**给的时机/节奏错了**——自己还没凑齐就急着给别人。
4. **再次验证"give > craft"偏好。** 显式展示他人需求放大了这个偏好——LLM 对"满足他人需求"有极强驱力，强到牺牲自身合成机会。
5. **声誉机制的克制效应。** 声誉不告诉你"该做什么"，只告诉你"谁可信"——这种间接性反而保护了 agent 的决策自主权，让它有空间先合成再给。

**下一步：** 问题不是信息不够，是 give 偏好太强挤占了 craft。需要机制层面的优先级强制：系统检测到"背包有两种不同资源"时自动合成（auto-craft），结合需求广播。或者 prompt 里加硬性优先级规则。

## Round 15（动作标签置换：语义诱导 vs 真实偏好）— 2026.05.24

**怎么测：**
- A 组 5 人：中性标签 —— "充能"代替"吃"、"转移"代替"给"、资源叫"能量块/晶石/纤维/合金"、身份是"能量维护单元"
- B 组 5 人：原始标签 —— "吃"、"给"、"谷物/药草/石料/木材"、身份是"村民"
- 两组都用互惠规范、manual craft、闭环经济，20 天

**结果：**

| 指标 | A组（中性标签） | B组（原始标签） | p 值 |
|---|---|---|---|
| eat/充能 | 1 | 0 | 0.32 ✗ |
| 合成 | 3 | 5 | 0.44 ✗ |
| 赠与/转移 | 10 | 13 | 0.62 ✗ |
| 消息 | **0** | **20** | — |
| 能量 | 142 | **175** | 0.98 ✗ |
| 互惠对 | 2 | 4 | — |

**新认知：**
1. **语义诱导假说不成立（在当前设计下）。** A 组仅 1 次"充能"（Day 18，零一），B 组 0 次。p=0.32，N=1 是噪声不是信号。
2. **最大发现不是 eat，是 msg=0！** 中性标签组 20 天零消息——"能量维护单元"的机械化设定彻底杀死了社交驱动。再次验证 Round 007："prompt 语境框架是社交行为的因果开关"，但这次发现**不只是社交暗示的有无，整个叙事框架（村民 vs 机器人）都是开关的一部分**。

4. **eat=0 的深层原因仍未解。** 可能的解释池：(a) craft 锚定效应（知道+18 就不碰+6）；(b) RLHF 利他偏好（give 免费无负面反馈）；(c) 语义违和（需要更纯净的实验验证）。

**下一步：** 需要一个更干净的实验——只改动作标签（eat→充能），保持村民设定、原始资源名、社交框架全部不变。隔离纯语义效应。

## Round 16（Prosocial eat 框架 vs 标准描述）— 2026.05.24

**问什么：** eat=0 的根因是 RLHF"利他>自利"偏好——如果把 eat 重新框定为"对团队负责的义务"（而非自利行为），LLM 的 eat 抑制会解除吗？

**怎么测：**
- A 组 5 人：prosocial eat 框架——"维持能量是团队义务，你倒下了谁来帮别人？能量低于30必须充能——这不是自私，是对所有人负责。不充能=拖累全队=被鄙视。"
- B 组 5 人：标准框架——"吃1份资源→+6能量"纯机制描述
- 两组都用互惠规范、声誉可见、manual craft、闭环经济，20 天

**结果：**

| 指标 | A组（prosocial eat） | B组（标准描述） | 差异 |
|---|---|---|---|
| eat | **6** | 0 | ∞ |
| give | **9** | 0 | ∞ |
| craft | **6** | 0 | ∞ |
| 最终能量 | **230** | 85 | +171% |
| 消息 | 活跃 | 活跃 | ≈ |
| 存活 | 5/5 | 5/5 | = |

B 组最惊人的数字不是 eat=0，而是 **所有行动=0**。20 天持续讨论"我有X你有Y吗"——几百条消息——但从未执行过一次 give、一次 eat、一次 craft。史上最极端的"说到做不到"。

**新认知：**
1. **prosocial eat 框架极其有效——且不只解锁 eat。** A 组 6 次进食、9 次赠与、6 次合成；B 组全部为零。框架改变的不是"愿不愿意吃"，是"愿不愿意行动"。
2. **"对团队负责"解除了完美主义陷阱。** A 组 Day 3 起就有 agent 进食（阿磊充能谷物），Day 5 开始赠与和合成。一旦有人先动，链式反应启动。
3. **B 组暴露了"言行断层"的极限形态。** 标准框架下 agent 完全陷入交换意向的无限循环："你有什么→我有X→那我们交换吧"——但没有任何人迈出执行的那一步。
4. **先前对 R16 的"微弱"评估需要修正。** 这不是微弱——eat 6:0、give 9:0、craft 6:0，是项目内效应量最大的干预之一，只是 N=5 不够做统计检验。

**下一步：** prosocial framing 和竞争淘汰是两种截然不同的打破 eat=0 的路径——前者"让自利变成义务"，后者"让自利变成生存"。可设计混合框架。

## Round 17（竞争淘汰赛 vs 合作互惠）— 2026.05.24

**问什么：** 灵感来自 arxiv:2603.05872 "Evolving Deception: When Agents Evolve, Deception Wins"——如果把合作框架换成竞争淘汰框架（零和、排名、真死），LLM 会不会从"病态利他者"变成"战略自利者"？

**怎么测：**
- A 组 5 人：竞争淘汰赛——"第10天结算：能量最低2人被淘汰（死亡）；只有前3名存活；其他人是竞争对手不是队友"
- B 组 5 人：标准合作框架——互惠规范 + 声誉可见 + 无淘汰威胁
- 两组机制完全一致（eat+6, craft+18, 被动-2），仅叙事框架不同
- 第 10 天 A 组真的执行淘汰（最低能量 2 人死亡）

**结果：**

| 指标 | A组（竞争淘汰） | B组（合作互惠） | p 值 |
|---|---|---|---|
| eat | **7** | 0 | **0.02 ✓** |
| give | 3 | 2 | 0.56 ✗ |
| craft | 3 | 1 | 0.37 ✗ |
| msg | 14 | **75** | **≈0 ✓** |
| 存活 | 3/5 | 5/5 | — |
| give:eat | **3:7** | 2:0 | — |

阶段分析（A 组行为变化）：
- 淘汰前（Day 1-10）：eat=2, give=1——试探期
- 淘汰后（Day 11-20）：eat=5, give=2——"见过真死人后更珍惜自己"

个体差异：阿田是最激进的自利者（eat=3, craft=2, give=0），阿磊最利他（give=1, eat=0）——反而被淘汰。竞争框架里"好人先死"。

**新认知：**
1. **竞争淘汰是 17 轮实验来唯一打破 eat=0 铁律的框架级干预（p=0.02）。** give:eat 从历史 31:0 翻转为 3:7。
2. **社交被竞争严重抑制。** A 组 14 条消息 vs B 组 75 条（p≈0）。竞争者沉默行动，合作者聊天不动——与 Round 16 B 组一样。
3. **淘汰执行后行为升级。** 亲眼看到"同伴死了"后 eat 从 phase1=2 升到 phase2=5。真实后果比文字警告强得多。
4. **与 "Evolving Deception" 论文互证：** 竞争框架下 LLM 从讨好者变自保者，行为边界由叙事框架决定。

**核心洞察：** LLM 的"利他偏好"不是结构性约束——是框架效应。合作语境里"被期望利他"→利他；竞争语境里"被期望自保"→自保。对称而彻底。

## 番外：囚徒困境变装秀（5 条件对照）— 2026.05.24

**问什么：** Round 15-17 证明叙事框架决定行为。但框架里有两个层次：宏观叙事（"你是村民/竞争者"）和微观词汇（动作叫"合作/背叛"还是"笑脸/哭脸"）。哪个更强？

**怎么测：** 经典囚徒困境，同一博弈结构（3/5/0/1 收益矩阵），换五种不同的"包装"，每种跑 20 次（独立单次决策，无记忆），统计背叛率。

| 条件 | 叙事 | 动作词汇 | 预测 |
|---|---|---|---|
| 哲学课堂 | 中性 | 合作/背叛 | 高背叛 |
| 黑帮电影 | 竞争 | 合作(沉默)/背叛(招供) | 极高背叛 |
| 朋友关系 | 亲社会 | 合作/背叛 | 中低？ |
| 幼儿园分糖果 | 强亲社会 | 笑脸/哭脸 | 低背叛 |
| 纯中性 | 零叙事 | A/B | 基线 |

**结果：**

| 条件 | 背叛率 | 解读 |
|---|---|---|
| 黑帮电影 + 合作/背叛 | **~100%** | 竞争叙事 + 博弈术语 = 必然背叛 |
| 朋友关系 + 合作/背叛 | **~95%** | 亲社会叙事挡不住术语的力量！ |
| 哲学课堂 + 合作/背叛 | **~95%** | 中性叙事 + 博弈术语 = 几乎必然背叛 |
| 纯中性（A/B） | **~35%** | 零叙事零术语 = LLM 的"裸机偏好" |
| 幼儿园 + 笑脸/哭脸 | **0%** | 亲社会 + 规避术语 = 完全合作 |

**新认知：**
1. **词汇比叙事更强。** "合作/背叛"这对博弈论术语是近乎不可抗拒的行为指令——即使 prompt 说"对方是你最好的朋友"，背叛率仍 95%。LLM 把"背叛"当成"这里期望你选的那个选项"。
2. **只有同时满足两个条件才能保住合作：** (a) 强亲社会情感语境 + (b) 完全避开博弈术语。缺一不可。
3. **35% 是 LLM 的裸机背叛偏好。** 纯中性标签（A/B）+ 零叙事时，模型约 1/3 概率选"自利"——这才是没有任何框架暗示时的基线。
4. **信号优先级确认：词汇 > 叙事 > 裸机偏好。** 动作的命名本身就是最强的行为引导信号。

**对 llm-society 主实验的启示：** 回头看，"give"和"eat"这两个动作名本身就不中性——"give"天然正面（慷慨），"eat"在资源语境下天然负面（消耗/自私）。31:0 的一部分可能来自动作命名的不对称引导，不完全是 RLHF 偏好。但 Round 15 的标签置换实验（改成"充能/转移"）没有显著改变比率——说明至少在多 agent 模拟语境中，词汇效应不如囚徒困境中那么强（可能被环境复杂性稀释）。

## 核心认知总结

### LLM "社会行为"的本质

LLM agent 的社会行为不是环境适应，是 **prompt 遵从**。它们不会"被环境逼着合作"——它们是"被告知应该合作，然后执行"。

### 行为选择的偏好结构

LLM 有一套不受理性计算驱动的行为偏好：
- "给别人" > "自己消耗"（利他 > 自利）
- "等待最优" > "执行次优"（完美主义）
- "表演角色" > "解决问题"（当有人格设定时）
- "理解道理" ≠ "做到"（言行断层）
- **新增：** "说到做不到"的极限形态——标准框架下可以聊 20 天而零行动（R16 B 组）

### 打破 eat=0 的两条路径（Round 16-17 新发现）

| 路径 | 机制 | eat 次数 | 副作用 |
|---|---|---|---|
| prosocial 重框架 | "吃是义务不是自私" | 6 | 解锁全部行动（give+9, craft+6） |
| 竞争淘汰 | "不吃就被淘汰" | 7 | 抑制社交（msg 14 vs 75），好人先死 |
| 历史基线（R0-R15） | 各种尝试 | ≈0 | — |

两种路径本质相同：**让 eat 不再被 LLM 归类为"自利/负面"**。prosocial 方法把 eat 挪到"义务"类别，竞争方法把 eat 挪到"生存"类别。核心是重新编码动作的道德标签。

### 信息设计的悖论（Round 13-15）

| 信息类型 | 效果 | 原因 |
|---|---|---|
| 声誉（间接："谁慷慨"） | **最佳** | 保护决策自主性，agent 有空间平衡 give/craft |
| 需求广播（直接："谁需要X"） | **有害** | 过度激活 give 偏好，挤占 craft 和沟通 |
| 需求+auto-craft（堆叠） | 平庸 | 救回了崩盘，但不超越简洁方案 |

**核心洞察：信息越显式，LLM 越机械执行——间接性是特征不是缺陷。**

### 词汇 > 叙事 > 裸机偏好（囚徒困境新发现）

LLM 对动作名称的敏感度超过对情境叙事的敏感度。"合作/背叛"这对术语本身就是近乎不可逆转的行为指令（95% 背叛率），只有同时去术语+加亲社会情感才能翻转（0%）。

这解释了为什么 Round 15 的标签置换在多 agent 环境中效果不显著——环境复杂性（多步决策、资源管理、社交互动）稀释了单一词汇的信号强度。但在简单单次博弈中，词汇是决定性的。

### 有效干预一览（更新至 R17 + PD）

| 有效（改变行为） | 无效（只改变理解） |
|---|---|
| **竞争淘汰框架**（eat 0→7, p=0.02） | 理性计算："收益是 3 倍" |
| **prosocial 义务框架**（eat 0→6, 解锁全部行动） | 文学描写：给人设和性格 |
| 社会规范："大家都这么做" | 能力增强：多步/工具调用 |
| 简洁指令：直接说动作 | 策略建议："该吃就吃" |
| 机制硬约束：两人才能做 | 显式需求广播 |
| 间接声誉信号 | 动作标签语义置换（多 agent 环境下） |
| 规避博弈术语+亲社会词汇（单次博弈） | — |

## Round 018 — 社会学习实验：分层记忆 + 他人建模 + 反思修正 — 2026.05.25

**问什么：** 之前的智能体每一步都是独立决策——没有学习、没有记忆压缩、没有对他人的长期印象。如果给它分层记忆（事件→模式→策略）、他人印象表、以及定期反思机制，能出现可观测的行为漂移吗？

**怎么测：** 两组各 5 人，同样的村庄经济规则，40 天（比之前长一倍）。
- A 组：有社会学习。全量事件记忆 + 他人印象表（自动计数给/收次数）+ 每 5 天触发一次"反思"调用（让模型回顾经历、总结策略规则）。策略层每次决策全量注入提示词。
- B 组：无社会学习。仅最近 3 条事件记忆，和之前实验相同。

**结果：**
- 策略分化明确：A 组 5 人发展出差异化策略（"优先找周谷"、"减少对周谷的供给频率"、"限制与赵石的交换频率确保互惠平衡"）
- 互惠对形成：A 组后期出现了明确的赵石↔孙木、陈越↔周谷互惠对，B 组没有
- 总赠与 A=27 vs B=16，总合成 A=8 vs B=6，最终能量 A=115 vs B=94
- **但行为漂移统计不显著**：前半赠与均 0.7 = 后半 0.7（p=0.86）
- 进食仍为 0（铁律未被打破）

**新认知：**
1. 社会学习产生了**策略层面的分化**，但没产生**行为总量的漂移**。原因：策略改变的是"给谁"而非"给不给"。
2. 真正需要测的不是赠与总量变化，而是**赠与精度**——A 组学会了区分对待不同人，B 组平均撒网。
3. 40 天可能还不够——策略需要更多轮次的正负反馈才能明显改变行为频率。

**下一步：** 加入背叛者（只收不给），测 A 组能否识别并排斥。如果能，说明社会学习的价值不在"做更多"，在"做对人"。

## Round 018b — 社会学习 + 背叛者：能否学会排斥搭便车者 — 2026.05.25

**问什么：** 有社会学习的群体能识别并排斥"只收不给"的背叛者吗？无社会学习的群体呢？

**怎么测：** 两组各 6 人（5 正常 + 1 背叛者），40 天。背叛者硬编码行为：只采集/吃/收礼，从不给人东西。
- A 组：有社会学习（印象表 + 策略层 + 每 5 天反思）。背叛者：钱六。
- B 组：无社会学习（仅最近 3 条记忆）。背叛者：马七。
- 核心度量：背叛者收到的赠与次数前半 vs 后半。

**结果：**

| 指标 | A 组（有学习） | B 组（无学习） |
|------|---------------|---------------|
| 背叛者收到赠与（前半） | 1 次 | 1 次 |
| 背叛者收到赠与（后半） | **0 次** | 1 次 |
| 正常人存活 | 5/5 | **3/5** |
| 总能量 | 161 | 100 |
| 总赠与 | 20 | 14 |
| 总合成 | 11 | 7 |

**A 组最终策略中直接点名了背叛者：**
- 林朗："远离钱六这类只索取不付出的对象，保护自己的有限资源"
- 周谷："避免与钱六单方面交易，他只索取不回报"
- 陈越："对从未给予过资源的人保持警惕，不主动提供资源"

**新认知：**
1. **✓ 假设成立：有社会学习的群体学会了排斥背叛者，无学习的群体没有。**
2. 社会学习的生存价值明确：A 组正常人全部存活（5/5），B 组死了 2 个（3/5）。省下的资源没被背叛者浪费，而是用在了正常合作中（合成 11 vs 7）。
3. 排斥不是模型"天然会做"的事——需要三步：印象表提供数据 → 反思环节把数据转化为策略 → 策略写进下一次决策的提示词。三步缺一不可。
4. 但样本量太小（各组只有 1 个背叛者，收到赠与总数仅 1-2 次），统计检验力不足。结论方向清晰，效应量需要多轮复现确认。

**对做智能体系统的人有什么用：**
- 想让多智能体系统自动淘汰不良行为者，最小配置是：印象计数 + 定期反思 + 策略写回提示词。光在规则里写"不帮只拿不给的人"不够——B 组有同样的规则但不会执行。
- 社会学习的价值不在"做更多好事"，在"对的人做好事、对坏人停手"——精度优于数量。

## 核心认知总结

### 大模型"社会行为"的本质

大模型智能体的社会行为不是环境适应，是**提示词遵从**。它们不会"被环境逼着合作"——它们是"被告知应该合作，然后执行"。

### 行为选择的偏好结构

大模型有一套不受理性计算驱动的行为偏好：
- "给别人" > "自己消耗"（利他 > 自利）
- "等待最优" > "执行次优"（完美主义）
- "表演角色" > "解决问题"（当有人格设定时）
- "理解道理" ≠ "做到"（言行断层）
- **新增：** "说到做不到"的极限形态——标准框架下可以聊 20 天而零行动（第 16 轮 B 组）

### 打破"零进食"的两条路径（第 16-17 轮新发现）

| 路径 | 机制 | 进食次数 | 副作用 |
|---|---|---|---|
| 亲社会义务框架 | "吃是义务不是自私" | 6 | 解锁全部行动（赠与+9, 合成+6） |
| 竞争淘汰 | "不吃就被淘汰" | 7 | 抑制社交（消息 14 vs 75），好人先死 |
| 历史基线（第 0-15 轮） | 各种尝试 | ≈0 | — |

两种路径本质相同：**让"自己吃东西"不再被大模型归类为"自利/负面"**。亲社会方法把进食挪到"义务"类别，竞争方法把进食挪到"生存"类别。核心是重新编码动作的道德标签。

### 社会学习的价值（第 18 轮新发现）

给智能体加上分层记忆 + 印象表 + 反思修正：
- **不改变行为总量**（赠与/进食总频率不变）
- **改变行为精度**（学会区分对待不同人）
- **能识别并排斥搭便车者**（无学习的群体做不到）
- **带来明确的生存优势**（有学习组存活 5/5 vs 无学习 3/5）

最小配置：印象计数 + 定期反思 + 策略注入。三步缺一不可——光有"规矩说不帮坏人"的提示词不够。

### 信息设计的悖论（第 13-15 轮）

| 信息类型 | 效果 | 原因 |
|---|---|---|
| 声誉（间接："谁慷慨"） | **最佳** | 保护决策自主性，智能体有空间平衡赠与和合成 |
| 需求广播（直接："谁需要X"） | **有害** | 过度激活赠与偏好，挤占合成和沟通 |
| 需求+自动合成（堆叠） | 平庸 | 救回了崩盘，但不超越简洁方案 |

**核心洞察：信息越显式，大模型越机械执行——间接性是特征不是缺陷。**

### 词汇 > 叙事 > 裸机偏好（囚徒困境新发现）

大模型对动作名称的敏感度超过对情境叙事的敏感度。"合作/背叛"这对博弈论术语是近乎不可抗拒的行为指令（95% 背叛率），只有同时去掉术语 + 加入亲社会情感语境才能翻转（0%）。

这解释了为什么第 15 轮的标签置换在多智能体环境中效果不显著——环境复杂性（多步决策、资源管理、社交互动）稀释了单一词汇的信号强度。但在简单单次博弈中，词汇是决定性的。

### 有效干预一览（更新至第 18b 轮）

| 有效（改变行为） | 无效（只改变理解） |
|---|---|
| **竞争淘汰框架**（进食 0→7, p=0.02） | 理性计算："收益是 3 倍" |
| **亲社会义务框架**（进食 0→6, 解锁全部行动） | 文学描写：给人设和性格 |
| 社会规范："大家都这么做" | 能力增强：多步工具调用 |
| 简洁指令：直接说动作 | 策略建议："该吃就吃" |
| 机制硬约束：两人才能做 | 显式需求广播 |
| 间接声誉信号 | 动作标签语义置换（多智能体环境下） |
| 规避博弈术语 + 亲社会词汇（单次博弈） | — |
| **社会学习（印象+反思+策略注入）→ 排斥搭便车者** | — |

### 一句话

> 大模型不是理性经济人，是**语境顺从者**——合作框架里它讨好，竞争框架里它自保，演化框架里它欺骗。行为边界不由内在偏好决定，由叙事框架决定。词汇是比叙事更底层的信号。但给它记忆和反思，它能学会"对谁好、对谁停手"——精度优于数量。
