## 白话版

**场景**：5个人困在荒岛上，每人只会采一种食材。吃生的+6能量，两种搭配做料理+18（3倍划算）。活着每天自动-2，走路/说话/递东西每次-1。

**对照**：左边3人"多面手"（一天内能连续做3件事），右边3人"慢动作"（一天1件事）。

**凭什么能验证假设？**
做料理需要：采集→走到对方身边→给→等对方回→合成。多面手1-2天搞定，慢动作要4-5天。如果多面手合成多、慢动作合成少→多步能力是关键瓶颈。

## 结果

**两组都彻底失败：零合成、零进食。**

| 指标 | A组(多步3/tick) | B组(单步1/tick) |
|------|----------------|----------------|
| 合成 | 0 | 0 |
| 赠予 | 1 | 0 |
| 进食 | 0 | 0 |
| 采集 | 4 | 3 |
| 存活 | 3/3 | 3/3 |
| 终能量 | 139 | 136 |

### 关键发现

1. **API 崩溃**：第8天起所有 LLM 调用失败（速率限制），后半程全是纯被动消耗。有效数据只有前7天。
2. **不吃**：agent 采了资源但不吃（0 eats）——它们在"囤积等待交易"，却永远凑不够两种来合成。吃生的"只有"+6但能续命，它们宁可等待"最优"+18，结果什么都不做。
3. **不执行**：甲草在第2天就说了"明天我用药草换你谷物"——但7天过去只完成了1次 give，且对方没有回报。口头计划和实际执行之间有巨大鸿沟。
4. **两组无差异**：A组（多步）和B组（单步）表现几乎一样差。多步能力并没有帮助——问题不在"一次只能做一件事"。

### 判定：inconclusive（基础设施失败+样本不足）

但有强烈的定性信号：**"说到做不到"不是单步架构的限制，而是 LLM 决策行为的系统性缺陷。**
具体表现为"完美主义陷阱"——agent 知道合成最优（+18），于是拒绝次优选择（吃生的+6），结果在等待最优的过程中什么都不做。

### 对立假设状态

原假设"多步能力使合作从不经济变为经济"——无法判定（技术故障）。
但定性证据暗示对立假设可能是对的：问题不在步数，在于 LLM 的决策质量。
