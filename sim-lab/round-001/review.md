## 蓝队（Claude）

### 评分：4/10

### 致命缺陷：不可证伪

1. **系统 prompt 直接指导合作**："合作采矿是最佳策略" + "建议：用 say 联系附近的人约定在矿石处会合"
   - 效果等同于"告诉 LLM 去社交"
   - 无论结果如何都无法区分"涌现" vs "执行 prompt 指令"

2. **无控制组**。缺少"资源充裕但有合作机制"的对照。只有 A 组高压 vs 无对照，结果不可解释。

3. **偶然 vs 协调不可区分**。20 agent 在 20×20，相邻概率 ~25%。随机游走也能产出非零 mine_success。缺少因果链追踪。

### 必改项

1. 删除策略建议。prompt 只描述机制规则，不推荐策略
2. 加控制组。同一次 run 内分 A 组（高压：独食少）和 B 组（低压：独食多）
3. 因果链标记。mine_success 记录过去 5 tick 内双方是否有消息往来 → 区分"协调采矿"和"偶遇采矿"

### 建议项

4. 双向 mine 确认（双方同 tick 都出 mine 才算）
5. 随机基线对比（不调 LLM，随机动作的 mine_success 期望值）
6. 消息内容分类（含"矿/合作/来" vs 废话）
