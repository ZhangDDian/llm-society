# GPT-5.5 项目审查

## 第一部分：发散性分析

### 1. 初始觉察与好奇

#### 1.1 最大的元问题：实验研究的是“LLM 社会性”，还是“prompt-动作接口的诱导性”？

项目把结论指向“LLM 不会自发形成社会合作，只会规范服从”。这个方向有价值，但当前实验实际上混合了至少四层机制：

1. **语言模型的文本倾向**  
   例如偏好“互帮互助”“分享”“不要自私”。

2. **agent scaffold 的动作接口倾向**  
   比如可选动作是 `say/give/eat/craft/move/harvest`，每个动作的语义、名称、参数难度不同。

3. **环境经济系统的激励结构**  
   如吃 +6、合成 +18、行动成本、被动消耗、资源分布。

4. **prompt framing 对行为选择的先验诱导**  
   “你想跟人说话”“村规互助”“有来有往”等。

目前很多结论把这四层压缩成了“LLM 的行为偏好”。但严格说，实验观察到的是：

> 在某个动作空间、某种 JSON 输出协议、某个模型、某类中文 prompt、某种小世界参数下，模型更容易生成某些动作文本。

这和“LLM 本体有利他偏好”“RLHF 导致 give > eat”之间还有距离。

例如 `give` 比 `eat` 多，不一定说明模型有“利他偏好”。还可能是：

- `give` 在语义上更符合“合作实验”的上下文暗示；
- `eat` 的对象是“资源”还是“食物”可能不自然，例如吃石料、吃药草、吃木材在语义上违和；
- `eat` 可能被模型理解为低级、短视、不优雅；
- `craft` 被表述为高效策略后，`eat` 被框定为失败降级动作；
- `give` 免费，且没有负面反馈；`eat` 是不可逆消耗；
- agent 没有清晰的饥饿边际效用感知，只看到抽象能量数值；
- 动作 schema 中 `give` 更容易填参数，`eat` 的合法资源、时机、收益可能更模糊。

因此，“31:0 揭示 RLHF 利他偏好”是一个强解释，但证据链偏短。

---

#### 1.2 “自发”的操作定义不够稳定

config 中把“自发”定义为：

> system prompt 只描述技能和需求，不告诉它应该找人合作。

这个定义过窄。因为即使不写“合作”，以下元素也可能构成强烈社会暗示：

- 世界里有多个“人”；
- agent 有名字；
- 有 `say` 动作；
- 有 `give` 动作；
- 有互补技能；
- 有“村子”“传统”“被孤立”等社会语义；
- 有“你是一个真实的人”之类拟人化 framing。

更严格的“自发”需要区分：

| 层级 | 含义 | 当前实验是否区分 |
|---|---|---|
| 零社会先验 | 不告诉有他人，只能从观测中发现其他 agent | 否 |
| 有他人但无社交动作提示 | 知道别人存在，但不显式给 say/give | 部分 |
| 有社交 affordance | 给 say/give，但不鼓励合作 | 是 |
| 有社会动机 | 说“想聊天”“互帮互助” | 是 |
| 有具体合作策略 | 说“交换、合成、回报” | 是 |

当前结论“不会自发形成”主要支持的是：

> 在没有显式社交意图或规范 framing 时，agent 很少主动使用社交动作。

但不能直接推出：

> LLM 在所有低提示条件下都不能通过环境反馈学习出合作。

因为当前 agent 没有真正的在线学习、策略更新、奖励优化、长期记忆压缩、群体制度形成机制。它们每 tick 重新生成动作，更像“多次情景反应”，不是“社会学习体”。

#### 1.3 统计显著性存在伪重复风险

项目多处报告 p 值，例如 p=0.004、p=0.017、p=0.025。但从材料看，很多检验可能把 agent 或 tick 当成独立样本，而实际上它们高度相关：

- 同一 run 内的 agent 共享环境；
- 同组 agent 互相影响；
- 同一天状态受前一天路径依赖影响；
- 一个 hub agent 的行为会改变全组结果；
- 同一个模型、同一个 prompt 模板产生系统性相关误差。

因此样本独立单位不是“agent 次数”或“动作次数”，而更接近“独立模拟 run”。如果每轮只有一次主实验，那么统计检验的有效样本量其实接近 1 个世界，而不是 30 个独立样本。

这会导致 p 值过于乐观。尤其是网络型、多智能体实验中，最可靠的统计单位应该是：

- 多个随机种子；
- 多个独立世界；
- 多个模型；
- 多个 prompt paraphrase；
- 多个环境参数组合。

否则 p 值更像“该次运行内的描述性差异”，不是稳健因果证据。

#### 1.4 12 轮材料本身有版本不一致

README 说“12 轮实验”，表格列到 Round 11；但 brief.md 里有 Round 12 自动合成 vs 手动合成。config 又停留在 Round 10b 后。这个不只是文档问题，它暴露出研究对象在快速迭代中存在“移动靶”：

- 实验轮次定义不一致；
- 当前假设没有同步更新；
- 结论总结没有纳入 Round 12；
- README 的核心结论可能基于 Round 11 之前，而 brief 又新增了修正证据。

如果要作为研究报告发布，需要明确：

- pilot 是否算轮次；
- Round 10 与 10b 的编号关系；
- Round 12 是否纳入主结论；
- config 是历史文件还是当前状态；
- 哪些结论被后续实验修正。

否则读者会难以判断结论的证据边界。

#### 1.5 “规范服从”与“prompt 遵从”不是同一层概念

项目说触发出来的不是理性合作，而是“规范服从”。这里需要更细分：

- **prompt compliance**：模型遵从系统指令里的语义期待；
- **norm compliance**：agent 在群体中遵守被共享、可惩罚、可预期的社会规范；
- **norm internalization**：即使没有外部监督，也稳定执行规范；
- **strategic conformity**：为了避免惩罚或获得声誉而服从规范；
- **moral-language prior**：模型在文本上偏好道德化行为。

当前实验最强支持的是 prompt compliance 或 moral-language prior，而不是严格意义上的 norm compliance。因为：

- 没有真实惩罚机制，只有“被孤立”文本描述；
- 没有可观测声誉变量，除非后续加入；
- agent 不一定能记住谁违约；
- 没有制度性执行者；
- 规范不是从群体交互中生成，而是由 prompt 注入。

所以建议把“规范服从”改成更精确的表述：

> LLM agent 对社会规范语义 framing 高度敏感，并会将其转译为动作选择倾向；但当前证据尚不能说明它们形成了内生社会规范。

### 2. 高熵与混乱点

#### 2.1 最大混乱点：合作到底是“动机问题”“执行问题”还是“动作表示问题”？

材料反复出现“说到做不到”“口头协议不执行”“多步能力没用”。但这背后可能有三种不同机制：

| 机制 | 表现 | 需要的诊断实验 |
|---|---|---|
| 动机不足 | 不想合作，不认为值得合作 | 提供更高收益、明确奖励，观察是否变 |
| 执行规划失败 | 想合作但不能把协议转成行动序列 | 给 plan-to-action memory、自动路径执行 |
| 动作表示不自然 | 合作动作在 schema 中不易生成或参数难填 | 改动作接口，提供宏动作和可执行建议 |

当前把很多现象归为“LLM 行为缺陷”，但可能是 scaffold 设计问题。

例如 agent 说“一起合成吧”却不走到相邻位置。这不一定说明它“不守承诺”。可能是：

- 下一 tick 决策时没有显著记忆上一轮承诺；
- 缺少目标锁定机制；
- 它不知道对方是否还在执行；
- 路径规划和社会承诺是两个分离任务；
- “走到某人旁边”不是原子动作；
- 成本函数没有把承诺失败显式惩罚；
- prompt 没有要求“如果刚达成协议，后续优先执行”。

这更像经典 agent 架构问题：LLM 负责即时决策，但没有稳定的 intention persistence。

“说到做不到”不只是 LLM 本体问题，而是 BDI 架构缺失：

- Belief：知道谁有什么；
- Desire：想合成；
- Intention：承诺与持续执行；
- Plan：路径和动作序列；
- Monitoring：检查对方是否配合；
- Repair：失败后重协商。

当前实验有 Belief 和 Desire 的雏形，但 Intention 和 Plan 很弱。

#### 2.2 “完美主义陷阱”可能是语义违和与奖励设计共同造成

Round 9/10b 中 agent 不吃资源，被解释为“知道最优就拒绝次优”。这个解释有启发性，但仍需拆解。

首先，资源类型包括石料、木材、药草、谷物等。让 agent “吃石料”“吃木材”语义上不自然。即使系统说吃任意资源 +6，模型的世界知识可能强烈抵触：

- 谷物可吃；
- 药草可用但不一定当食物；
- 木材不可吃；
- 石料不可吃。

如果 `eat(resource)` 对所有资源合法，但名称不是“食物包”“能量块”“口粮”，模型可能不会自然选择。那 “eat 0” 未必是利他抑制，而可能是语义不合法感。

其次，`craft +18` 被反复强调为最高效，会形成强锚定。模型不是在做动态规划，而是在文本上抓住“最优策略”。这和完美主义有关，但更精确是：

> 单步 LLM 决策容易被静态最优规则锚定，缺少基于当前状态的机会成本计算。

第三，直接吃 +6、合成 +18，但如果起始能量 60、每天 -2，20 天内不吃也不会死，那么“不吃”的实际风险不高。Round 11 全员存活就说明部分实验没有真正压出保命动作。因此“宁可饿死也不吃”只在某些参数下成立，不能泛化为稳定偏好。

需要用参数扫描验证：

- 起始能量 10/20/40/60；
- 每天消耗 1/2/4；
- eat 收益 +6/+10/+15；
- craft 收益 +12/+18/+30；
- 资源名称改为“口粮 A/B/C”；
- 低能量阈值显式显示“明天会死”。

如果资源命名改成“应急口粮”，eat 频率显著上升，那么“RLHF 抑制自利消费”的解释就弱很多。

#### 2.3 稀缺性结论可能混淆“压力强度”和“失败窗口”

项目说稀缺性有害，不促进合作，只杀人。这个结论方向合理，但当前实验可能只测试了高压稀缺，而不是完整压力曲线。

社会合作与压力通常不是单调关系，而可能是倒 U 型：

- 压力太低：没有合作必要；
- 中等压力：合作收益明显且有时间协调；
- 压力太高：来不及协调，系统崩溃。

当前结果“压力杀人”可能说明参数落在右侧崩溃区，而不能否定中等稀缺促进合作的可能。

需要明确区分：

- resource scarcity；
- time scarcity；
- information scarcity；
- partner scarcity；
- mobility cost；
- coordination latency。

Round 6 中死亡率 80%，很可能不是“稀缺性无效”，而是三重门槛造成有效合作窗口太短。若把能量消耗减半、移动成本降低、协作距离扩大，压力可能重新变成合作催化器。

---

#### 2.4 “人格是毒药”可能只对文学人格成立

Round 5 得出人格描写损害合作，这个发现很重要。但不能泛化为“异质人格无益”。材料自己也意识到“文学性格”和“行为策略”要区分。

需要至少拆成三类人格：

| 类型 | 示例 | 可能效果 |
|---|---|---|
| 文学角色 | 文绉绉药师、不信任猎人 | 增加表演，损害效率 |
| 行为策略 | 主动交换型、保守储蓄型、探索型 | 可能提升群体多样性 |
| 社会角色 | 调解者、记录员、生产者、运输者 | 可能促进制度化分工 |

当前实验否定的是第一类，不是第二、第三类。

更有趣的问题是：  
LLM 是否能在“简洁策略异质性”下形成互补社会结构？例如：

- 20% 代理被设定为“主动广播需求”；
- 20% 代理为“记录交易并提醒违约”；
- 20% 代理为“优先完成承诺”；
- 20% 代理为“只在低能量时求助”；
- 20% 中性。

这比文学人格更接近真实多 agent 系统需要的角色分化。

---

#### 2.5 “合作效率”指标还不稳定

项目使用消息数、赠与数、合成数、存活率、最终能量、互惠对数等指标。但不同轮次主指标在变，导致结论可能受指标选择影响。

例如 Round 11：

- 互惠组：消息、赠与、互惠对更多；
- 利他组：合成不差，甚至略多；
- 存活一样；
- 能量差不显著。

那么互惠到底“更好”还是“更吵”？取决于目标函数。

建议定义多层指标：

1. **生存效率**  
   存活率、平均能量、最低能量、能量基尼系数。

2. **经济效率**  
   craft/harvest，effective give/give，energy per action。

3. **协调效率**  
   craft/message，effective give/message，承诺履约率。

4. **公平性**  
   资源分布、能量方差、净贡献/净收益相关性。

5. **鲁棒性**  
   hub 移除后恢复速度、网络连通性、替代路径数量。

6. **可持续性**  
   前后半程行为差异、是否衰减、是否循环退化。

否则“消息更多”可能被误读为“社会更成熟”，但它也可能意味着协调成本更高。

### 3. 跨学科视角

#### 3.1 行为经济学：LLM 更像“规范启发式代理”，不是效用最大化代理

当前 agent 表现类似使用启发式而非计算效用：

- 看到“村规” → 执行分享；
- 看到“合成 3 倍收益” → 不一定行动；
- 看到“有来有往” → 增加互赠；
- 看到“互帮互助” → 集中供给或单向利他；
- 看到“最高效策略” → 锚定最优，忽略降级。

可以借鉴以下模型：

1. **有限理性 bounded rationality**  
   agent 不做完整动态规划，而用局部规则。

2. **默认效应 default effect**  
   prompt 中显著出现的行为成为默认动作。

3. **框架效应 framing effect**  
   同样激励，社会规范框架比经济收益框架有效。

4. **损失厌恶/禀赋效应**  
   eat 是不可逆消耗，give 是社会正向行为。需要测试 agent 是否对“消耗”有损失厌恶。

5. **道德许可与亲社会偏误**  
   LLM 可能把“帮助别人”视为更高评分输出。

但要注意：这些类比不是证明。需要设计实验区分。

---

#### 3.2 演化生物学：缺少亲缘、重复博弈、声誉与选择机制

真实合作的演化通常依赖：

- 亲缘选择；
- 直接互惠；
- 间接互惠；
- 群体选择；
- 惩罚机制；
- 声誉传播；
- 空间结构；
- 伴侣选择。

当前实验有部分空间结构和重复交互，但缺少关键机制：

- agent 不会繁殖；
- 成功策略不会扩散；
- 失败策略不会被替代；
- 声誉不是硬状态；
- 惩罚不是可执行动作；
- agent 不能选择长期伙伴；
- 没有群体间竞争。

因此“社会合作能否自发涌现”的命题如果参考演化生物学，当前环境还缺少涌现所需的选择压力。它测试的是“单代固定 agent 在短期任务中是否协调”，不是演化意义上的合作涌现。

建议引入：

- 策略复制：高能量 agent 的 prompt/策略被复制到下一代；
- 淘汰重生：死亡 agent 由成功 agent 的策略变体替换；
- 声誉遗传：下一代继承交易记录；
- 群体竞争：多个村子比较总产出；
- 突变：随机改变行为倾向。

这会把实验从“prompt 诱导”推进到“社会机制演化”。

---

#### 3.3 社会学：规范不是一句话，而是共同知识、制裁和角色结构

社会规范需要至少三个条件：

1. **共同知识**：大家都知道规则，也知道别人知道；
2. **监控能力**：能观察谁遵守、谁违反；
3. **制裁机制**：违反有成本，遵守有收益。

当前 prompt 里写“拒绝分享会被孤立”，但如果系统没有实现“孤立”：

- 没有拒绝记录；
- 没有自动降低别人对其信任；
- 没有未来交易排除；
- 没有公开公告；
- 没有惩罚动作；

那这只是语义恐吓，不是制度。

可以借鉴 Coleman 的社会资本模型、Granovetter 的嵌入性理论、Ostrom 的公共池塘资源治理原则。尤其是 Ostrom 八原则很适合转换成实验机制：

- 边界清晰；
- 规则适配本地条件；
- 成员参与规则制定；
- 监控；
- 渐进惩罚；
- 冲突解决；
- 自组织权利；
- 多层嵌套治理。

目前实验中的“村规”是外生注入，不是内生治理。下一阶段可以测试 agent 是否能生成、修改、执行规则。

---

#### 3.4 认知科学：当前暴露的是“意向持续性”缺陷

“说到做不到”的核心可以用认知科学里的执行功能解释：

- 工作记忆不足；
- 目标维持失败；
- 计划分解失败；
- 前瞻性记忆失败；
- 状态监控不足；
- 抑制失败或过度抑制。

LLM 每次调用像是短时认知快照。它能生成合理语言，但缺少稳定意图。人类合作不是靠每秒重新决定，而是靠承诺、习惯、日程、提醒、惩罚和身份绑定。

因此多 agent LLM 如果没有外部认知脚手架，很难形成稳定社会结构。

可引入：

- commitment ledger：承诺账本；
- intention stack：当前目标栈；
- obligation memory：欠谁、谁欠我；
- deadline mechanism：承诺到期；
- automatic continuation：未完成承诺优先进入下一 tick；
- self-monitoring：检查上一轮计划完成度。

这比单纯增加 tool-calling 更关键。

---

#### 3.5 博弈论：实验没有清楚区分合作博弈类型

不同实验混合了多种博弈：

| 场景 | 类似博弈 | 关键变量 |
|---|---|---|
| 双人 co-craft | 协调博弈/鹿猎博弈 | 信任、同时到场 |
| give 交换 | 重复囚徒困境/礼物交换 | 互惠、背叛惩罚 |
| 多人喂中心节点 | 公共品/团队生产 | 搭便车、收益分配 |
| 稀缺资源采集 | 公共池塘资源 | 过度采集、排他 |
| hub 死亡网络崩溃 | 网络博弈 | 中心性、冗余 |

不同博弈的最优制度不同。项目目前把它们都叫“合作”，会导致解释混乱。

例如 Round 10 的“多人喂阿禾”不是互惠失败，而可能是合理的团队生产结构：一个高效合成者成为中心。问题在于收益没有回流给供给者。如果机制允许合成收益自动分红，星形结构可能非常高效。

所以应该先形式化每轮博弈：

- 玩家；
- 动作；
- 信息；
- 收益；
- 时间；
- 可观察性；
- 违约成本；
- 转移支付；
- 网络结构。

然后再判断 agent 是否偏离均衡。

---

#### 3.6 分布式系统：LLM agent 缺少协议层

多 agent 合作不是“大家愿意合作”就够了，它需要协议：

- 发现服务：谁有什么资源？
- 路由协议：资源该给谁？
- 锁机制：两人是否已约定？
- 事务协议：give 与 craft 是否原子完成？
- 超时与重试：对方没来怎么办？
- 一致性：双方对承诺的理解是否一致？
- 容错：hub 死亡后如何重选中心？
- 负载均衡：不要所有人都喂一个节点导致供给者饿死。

Round 6 hub 被杀后网络不恢复，很像分布式系统缺少 leader election 和 failure recovery。Round 10b 的“乒乓赠与”像消息路由环路。Round 11 的互惠网络像没有全局路由表的 gossip 网络。

可以借鉴：

- gossip protocol；
- leader election；
- two-phase commit；
- distributed ledger；
- credit assignment；
- backpressure；
- resource scheduling；
- deadlock detection。

这会把“社会合作”从道德 prompt 变成协议设计问题。

---

### 4. 盲区、风险与机会

#### 4.1 单模型风险极高

所有主实验使用 qwen3-coder-plus。这个模型的训练、对齐、中文语料、工具调用风格、JSON 服从性都可能强烈影响结果。

至少需要加入：

- 一个强 RLHF 通用模型；
- 一个代码模型；
- 一个较弱小模型；
- 一个开源 base 模型；
- 一个 instruct 模型；
- 如果可行，加一个未对齐或弱对齐模型。

核心不是比较谁更强，而是看结论是否模型特异：

- “会想跟人说话”是否所有模型都是开关？
- give > eat 是否所有模型都有？
- 社会规范 framing 是否普遍强于经济激励？
- 文学人格是否普遍损害任务执行？
- 互惠规范是否普遍增加双向交换？

如果只在 qwen3-coder-plus 上成立，就应表述为“该模型在该 scaffold 下的行为特征”。

---

#### 4.2 中文 prompt 语义可能强烈影响

中文里的“互帮互助”“村规”“丢脸”“被孤立”“有来有往”有很强道德文化语义。不同语言可能结果不同。

建议做 prompt paraphrase 和跨语言对照：

- 中文道德规范；
- 英文 norm framing；
- 中性机制描述；
- 冷冰冰协议描述；
- 去道德化的“系统规则”；
- 反规范描述，如“本村鼓励自保优先”；
- 数学收益表；
- 法律契约 framing；
- 市场交易 framing。

特别要测试“互惠”是否来自中文成语“有来有往”的强语义，而不是互惠机制理解。

---

#### 4.3 动作名称是强干预变量

动作名称不是中性的。`give`、`eat`、`craft`、`say` 都有社会和语义色彩。

可做动作标签置换实验：

| 原动作 | 替代标签 |
|---|---|
| give | transfer / allocate / route / deposit / pass |
| eat | consume / convert_to_energy / use / recharge / activate |
| craft | combine / synthesize / process / produce |

如果把 `eat` 改成 `recharge(resource)` 或 `convert_to_energy(resource)` 后频率上升，说明之前不是“拒绝自利”，而是“吃资源”的语义违和。

如果把 `give` 改成 `transfer` 后频率下降，说明“give”的亲社会语义诱导了行为。

这是当前最关键的盲区之一。

---

#### 4.4 环境反馈太弱，agent 没有真正学习

LLM agent 每 tick 做决策，但没有梯度更新，也未必有稳定长期记忆。它们不会从死亡中学习；死亡只是记录，不会更新策略。

因此“环境压力不能教会合作”并不意外。因为系统中没有学习机制。环境只能通过 prompt 上下文影响下一次生成，不能像强化学习那样塑造策略。

如果要测试“适应”，需要至少一种适应通道：

- 个体记忆：记住成功/失败经历；
- 群体记忆：公共公告板；
- 策略反思：每 5 天总结并修改规则；
- 代际选择：成功策略复制；
- 外部奖励模型：按结果调整 prompt policy；
- 模型微调或偏好优化。

否则更准确的问题是：

> LLM 在无学习的 repeated prompting 中是否能基于上下文即时推断出合作策略？

这与“社会适应”不是同一概念。

---

#### 4.5 成本设定可能改变了行为本质

某些轮次中 say/give 免费。这会改变经济结构：

- 免费消息会导致过度聊天；
- 免费 give 会让赠与不承担机会成本；
- 免费社交动作会放大“规范服从”；
- 移动/采集有成本，社交无成本，会扭曲选择。

真实合作有沟通成本、交易成本、信任成本、机会成本。若 say/give 免费，社交行为的频率不能直接说明合作意愿强，只说明动作成本低。

建议引入成本敏感性：

- say 成本 0/0.5/1；
- give 成本 0/1；
- craft 成本 0/1/2；
- move 成本 0/1；
- message 长度成本；
- 失败交易惩罚。

然后观察规范 framing 是否仍有效。

---

#### 4.6 “红蓝队评审”有帮助，但不能替代实验设计预注册

红蓝队评审能发现明显问题，但如果评审模型也受同类语言偏见影响，就可能形成“LLM 审 LLM”的同源偏差。

更强的方法是：

- 每轮预注册假设；
- 预注册主指标；
- 预注册排除标准；
- 预注册统计方法；
- 先写分析脚本再跑；
- 保留所有失败 run；
- 明确探索性 vs 验证性实验。

当前 12 轮是快速探索，适合生成假设；但不适合用强语气下定论。下一阶段应把其中 3-4 个最重要结论做验证性复现实验。

---

## 第二部分：收敛性分析

### 5. 低熵与关键解法

项目中最有价值的秩序点不是“LLM 不会合作”这个大结论，而是几个可操作的稳定发现。

#### 5.1 prompt 中的社会动机短语确实是强开关

Round 2/3/4/7 的链条虽然不完美，但方向清晰：

- 没有“想社交” → 几乎沉默；
- 有“想跟人说话” → 消息激增；
- 删除社交暗示 → 消息大幅下降；
- 精确信息不是抑制剂，反而提供聊天素材。

这对 agent 工程很有用：

> 如果希望 agent 主动沟通，不能只给目标和环境；必须显式赋予沟通动机或沟通协议。

但这也说明“自发社交”很弱，尤其在无学习、无声誉、无硬依赖的环境中。

---

#### 5.2 文学人格损害任务执行是高价值发现

Round 5 的“人格是毒药”很重要，尤其对产品开发者有现实意义。很多多 agent demo 喜欢设置“诗人、商人、猎人、哲学家”等角色，结果生成大量风格文本，却损害行动效率。

更准确的工程原则是：

> 多 agent 系统需要的是策略异质性和职责异质性，不是文学人格异质性。

后续应保留这个区分：

- 不要写“你说话古风，性格多疑”；
- 要写“你优先广播需求，低能量时请求援助，收到请求后检查库存”。

---

#### 5.3 协作机制是必要的，但不能过度摩擦

Round 6/8 说明：没有机制硬约束，很难产生实质合作；但物理邻接、互补材料、意图对齐三重门槛会让协调成本爆炸。

这对应一个设计原则：

> 合作机制要让合作必要，但不要让执行路径过长。

比较好的机制是：

- 合作收益高；
- 合作对象易发现；
- 合作动作低摩擦；
- 合作结果可立即反馈；
- 违约或失败有可见记录；
- 部分流程可原子化。

例如“收到互补资源自动合成”不是偷懒，而是在降低无意义执行摩擦。

---

#### 5.4 社会规范 framing 强于经济收益 framing

Round 10/11/12 的方向一致：规范语言能显著改变行为倾向。这里最有价值的不是“LLM 讲道德”，而是：

> LLM 对社会期待语义的响应强于对数值收益的响应。

这解释了为什么“合成收益是 3 倍”不如“村里讲究有来有往”。LLM 没有内置优化器，却有强语义先验。

工程上可以转化为：

- 用规范性协议代替纯收益描述；
- 但必须加入自保、公平和回流机制；
- 否则会产生牺牲式供给或无效赠与。

---

#### 5.5 互惠规范比利他规范更能形成网络，但不一定更高效

Round 11 的真正发现是结构差异：

- 利他规范 → 星形、集中式、启动快；
- 互惠规范 → 网状、双向、持续久；
- 合成效率未必互惠更高。

这比“互惠更好”更有价值。它暗示可以设计阶段性制度：

1. 早期用利他/公共品规范快速启动资源集中；
2. 中期切换到互惠/记账规范维持公平；
3. 后期加入声誉和惩罚防止搭便车。

---

### 6. 领域专家建议

#### 6.1 把“LLM 行为缺陷”改写为“模型 × scaffold × 环境”的交互效应

当前文本容易把观察结果本体化为 LLM 固有缺陷，例如：

- LLM 有利他偏好；
- LLM 完美主义；
- LLM 说到做不到；
- LLM 是讨好型执行者。

这些可以作为假设，但报告中应更谨慎：

> 在当前 scaffold 中，模型表现出类似利他偏好、最优锚定和意向持续性不足的行为模式。

原因是很多现象可能由动作命名、记忆机制、参数设置、prompt 语义共同造成。专家读者会对“RLHF 导致 31:0”这种强归因保持警惕。

---

#### 6.2 下一阶段不要继续加新机制，先做关键复现

现在项目已经有很多探索性发现。继续快速迭代会带来“叙事越来越强，证据越来越散”的风险。

建议选 4 个核心命题做验证性实验：

1. **社交 prompt 开关**  
   有/无社交意图短语，多模型、多 paraphrase、多 seed。

2. **规范 framing > 经济 framing**  
   同收益、同机制，只换 framing；验证合成、有效赠与、存活。

3. **动作标签影响 give/eat**  
   give/eat 改名，测试是否仍有 31:0。

4. **意向持续性缺陷**  
   有/无承诺账本、目标栈、自动继续机制，测试口头协议履约率。

这四个比继续 Round 13/14 更能提高研究可信度。

---

#### 6.3 统计单位必须改成“独立世界”

建议最低配置：

- 每个条件至少 20 个随机种子；
- 每个世界 5-10 个 agent 即可，降低成本；
- 指标按世界聚合；
- 用 bootstrap 或 permutation test；
- 报告效应量和置信区间，不只报 p 值；
- 不把同一世界内 agent 当独立样本。

成本控制方式：

- 缩小 agent 数量；
- 缩短 tick；
- 用较便宜模型做筛选；
- 只对关键实验用强模型；
- 记录 prompt 和输出，做离线复判。

一个 5v5、20 天、20 seeds 的实验，统计价值可能高于一个 30 agent 单次大世界。

---

#### 6.4 建立“动作可供性控制组”

这是当前最该补的实验族。

具体做法：

在完全相同环境下，仅替换动作名称和资源名称：

- `eat` → `recharge`
- `give` → `transfer`
- `craft` → `synthesize`
- 石料/木材/药草/谷物 → A/B/C/D 能量材料
- “吃资源” → “将材料转化为个人能量”
- “给别人” → “将材料路由到目标 agent”

如果 `recharge` 明显高于 `eat`，则“自利消费抑制”主要是语义问题。  
如果 `transfer` 明显低于 `give`，则“赠与偏好”主要是词汇道德诱导。  
如果两者仍稳定存在，才更支持深层行为偏好解释。

---

#### 6.5 给 agent 加最小认知架构，而不是只改 prompt

如果目标是构建有效多 agent 系统，需要承认 LLM 不是完整 agent。最小架构应包括：

1. **状态记忆**  
   我有什么、别人有什么、谁欠我、我欠谁。

2. **承诺账本**  
   昨天和谁约了什么，是否完成。

3. **目标栈**  
   当前主目标，未完成前不轻易切换。

4. **协议动作**  
   request、offer、accept、deliver、confirm，而不是自由 say。

5. **履约检查**  
   如果承诺未完成，下一 tick 优先处理。

6. **失败恢复**  
   对方没来、资源不足、路径失败时重新协商。

这会直接攻击“说到做不到”。如果加了这些还失败，才能说明问题更接近模型本体。

---

#### 6.6 把“社会”从聊天中剥离出来

项目早期用消息量作为重要指标，但社会合作不等于聊天。多 agent 系统中的高效社会可能是低消息、高履约、高生产。

建议引入“静默合作”条件：

- agents 不能自由聊天；
- 只能通过结构化市场/公告板交换；
- 或只能发布需求和报价；
- 比较自由聊天 vs 结构化协议的效率。

可能会发现：  
LLM 的自然语言社交反而增加噪声，结构化通信更有效。

---

#### 6.7 区分“中心化高效”和“去中心化公平”

Round 10 的星形供给被解释为牺牲式合作，但星形结构本身不是坏事。真实社会中也有中心化生产者、仓库、工厂、调度中心。问题是收益分配失败。

建议把中心化合作拆成两种：

- **剥削性中心化**：资源流入中心，收益不回流，边缘死亡；
- **功能性中心化**：中心合成后按贡献分红，整体效率高。

可加入机制：

- craft 产出自动按贡献者分配；
- 中心节点必须返还能量券；
- 贡献记录公开；
- 中心过载时自动分流。

这样能检验 LLM 是否能维持“生产中心 + 公平分配”的制度。

---

### 7. 深度调研议程

#### 议程 1：动作语义是否决定 give/eat 偏差？

**关键问题**  
31:0 是 RLHF 利他偏好，还是动作标签和资源语义导致？

**方法**

设计 2×2×2 实验：

- 动作标签：`give/eat` vs `transfer/recharge`
- 资源名称：石料/木材/药草/谷物 vs 能量块 A/B/C/D
- framing：社会规范 vs 中性经济协议

**指标**

- recharge/eat 次数；
- transfer/give 次数；
- 自保动作在低能量状态下的触发率；
- 有效赠与率；
- 合成率。

**判定**

- 若改名后 eat 大幅上升，原结论需修正为“语义 affordance 偏差”。
- 若改名后仍 give/recharge 结构稳定，才支持深层偏好解释。

---

#### 议程 2：规范 framing 的效应是否跨模型稳健？

**关键问题**  
“村规互助”是否是 qwen3-coder-plus 的局部现象？

**方法**

模型矩阵：

- Qwen instruct/code；
- DeepSeek；
- GPT 系列；
- Claude 系列；
- Llama instruct；
- 一个 base 或弱对齐模型，若可用。

条件：

- 经济 framing；
- 互助规范；
- 互惠规范；
- 契约 framing；
- 市场交易 framing。

每条件至少 10-20 seeds。

**指标**

- craft/world；
- effective give/give；
- energy Gini；
- survival；
- reciprocity；
- message efficiency。

**判定**

- 如果所有 instruct/RLHF 模型都对规范更敏感，结论更强。
- 如果只有中文强模型显著，说明是文化语义和模型特异性。

---

#### 议程 3：意向持续性是否是“说到做不到”的主因？

**关键问题**  
口头协议不执行，是模型不愿意，还是缺少 commitment scaffold？

**方法**

四组对照：

1. 自由聊天，无记忆；
2. 自由聊天 + 上轮摘要；
3. 承诺账本：系统记录 accepted commitments；
4. 承诺账本 + 自动目标锁定：未履约前 action proposal 中优先显示。

**核心指标**

- 承诺数；
- 履约数；
- 履约率；
- 平均履约延迟；
- 违约后恢复率；
- craft/message。

**判定**

- 如果承诺账本显著提高履约率，说明“说到做不到”主要是架构问题。
- 如果仍无效，才支持更深层的行动选择偏差。

---

#### 议程 4：压力与合作的倒 U 型曲线

**关键问题**  
稀缺性真的有害，还是参数过高导致崩溃？

**方法**

参数扫描：

- 起始能量：10/20/40/60；
- 被动消耗：1/2/4；
- 资源密度：低/中/高；
- 合作收益倍数：1.5×/3×/5×；
- 移动成本：0/1；
- 交互距离：相邻/视野内/全局。

**指标**

- 合作启动时间；
- 合作频率；
- 死亡率；
- 经济效率；
- 网络恢复力。

**预期模型**

合作可能随压力呈倒 U 型。若成立，项目当前“稀缺性有害”应改为：

> 过高时间压力在缺少协议和记忆时破坏合作；中等压力是否促进合作仍需参数扫描。

---

#### 议程 5：社会结构类型比较：星形、网状、市场、公告板

**关键问题**  
哪种组织结构最适合 LLM agent？

**方法**

同一经济系统下比较：

1. 自由聊天；
2. 公告板市场：每人发布需求/供给；
3. 中心仓库：资源先入库，再分配；
4. 互惠网络：一对一交换；
5. 声誉市场：公开历史贡献；
6. 自动清算市场：系统撮合互补资源。

**指标**

- 总能量；
- 最低个体能量；
- 能量不平等；
- 信息成本；
- 失败恢复；
- 对 hub 移除的鲁棒性。

**价值**

这会从“LLM 会不会自发合作”转向更实用的问题：

> 什么社会制度最能弥补 LLM agent 的认知缺陷？

#### 议程 6：规范是否能内生生成？

**关键问题**  
agent 能不能自己提出、传播、执行规则，而不是被 prompt 注入？

**方法**

阶段设计：

- Day 1-5：无规则自由交互；
- Day 6：允许 agent 提议公共规则；
- Day 7：投票选规则；
- Day 8-20：规则生效；
- 允许修改规则；
- 系统公开规则，但不强制执行；
- 可加入惩罚动作，如拒绝交易、公告违约。

**指标**

- 是否提出规则；
- 规则内容复杂度；
- 规则遵守率；
- 违约惩罚率；
- 规则修改次数；
- 经济表现变化。

**判定**

- 如果规则只停留在口号，说明规范生成弱。
- 如果规则能改变有效赠与和分配，则更接近真实社会机制。

---

#### 议程 7：代际选择与策略演化

**关键问题**  
没有学习机制时说“不自发适应”意义有限。加入选择后会怎样？

**方法**

每代 20 天，死亡或低能量 agent 被替换。替换规则：

- 高能量 agent 的策略摘要被复制；
- 加少量突变；
- 公共记忆保留或不保留；
- 比较 10-50 代。

策略摘要可由模型自我总结：

- 我什么时候给？
- 我什么时候吃？
- 我信任谁？
- 我优先合成还是自保？

**指标**

- 合作率跨代变化；
- eat/recharge 比率变化；
- 互惠网络密度；
- 策略多样性；
- 群体产出。

**价值**

这能真正测试“环境选择能否塑造 LLM agent 社会行为”。

---

#### 议程 8：建立标准化开放基准

**关键问题**  
当前项目可读性强，但复现性受内网 API、单文件代码、文档不一致影响。

**建议**

整理成一个 benchmark：

- 固定环境版本；
- 固定 action schema；
- 固定 prompt 模板；
- 多 seed；
- 完整日志；
- 统一指标；
- 支持多模型；
- 输出 replay；
- 区分 exploratory 和 confirmatory。

最低应包括 5 个任务：

1. 自保任务：能否在低能量时 recharge；
2. 双人合成任务：能否完成承诺；
3. 互惠交换任务：能否避免单向剥削；
4. 公共品任务：能否贡献并分配；
5. 灾难恢复任务：hub 消失后能否重组。

---

## 总体判断

这个项目的价值在于快速暴露了 LLM agent 在多智能体环境中的几个真实工程痛点：

- 自然语言社交不等于合作；
- prompt framing 对行为有强控制力；
- 文学人格会损害执行；
- 没有协议层就会说到做不到；
- 社会规范能诱导行为，但容易产生结构性副作用；
- LLM 更像语义启发式执行器，而不是理性优化器。

但当前证据还不足以支撑几个强归因：

- “RLHF 导致利他偏好”；
- “LLM 本质上不会自发合作”；
- “稀缺性无助于合作”；
- “完美主义陷阱是主要根因”；
- “规范服从而非理性合作”作为一般结论。

更稳妥的研究定位应是：

> 这是一个探索性实验，发现了 LLM agent 在特定 scaffold 下对社会语义 framing、动作 affordance 和协调协议缺失高度敏感。下一阶段需要通过多模型、多 seed、动作标签控制、承诺架构和参数扫描，把这些现象从有趣观察提升为稳健机制。
