# 模拟进度条展示规则

## 场景
异步任务，预期 ~8min 完成，实际时长不确定（可能 3min 也可能 15min+）。
后端只给两个信号：**开始** 和 **完成**，前端模拟进度。

## 公式

```
前 5 秒：progress = t / 5000 × 15          （线性冲到 15%）
5 秒后：progress = 15 + 84 × (1 − e^(−3 × t / T))
```

- `t` = 已过时间（ms）
- `T` = 预期时长（默认 480000ms = 8min）
- 输出硬上限 99%

## 规则

| # | 规则 | 说明 |
|---|------|------|
| 1 | 永不回退 | 当前帧 ≥ 上一帧，违反则取上一帧值 |
| 2 | 永不停转 | 每 tick 至少 +0.01%，视觉上始终在动 |
| 3 | 上限 99% | 模拟值硬顶 99%，100% 仅由后端完成信号触发 |
| 4 | 完成平滑 | 收到完成信号后 ease-out 过渡到 100%，时长 = `max(300, (100 − current) × 8)` ms |
| 5 | 不显示剩余时间 | 不确定的倒计时比没有更焦虑 |
| 6 | 超时兜底 | 超过 1.5T（12min）未完成，进度在 98%~99% 间以 0.01%/2s 缓慢爬升，文案切为「仍在处理中…」|

## 更新频率

| 参数 | 值 | 说明 |
|------|----|------|
| tick 间隔 | 200ms | 定时器周期 |
| 每帧最大步进 | 0.5% | 防止视觉跳跃 |
| 每帧最小步进 | 0.01% | 保证呼吸感 |

## 进度-时间对照

以公式计算，T = 8min：

| 时间 | 进度 |
|------|------|
| 0s | 0% |
| 5s | 15% |
| 1min | 41% |
| 2min | 59% |
| 4min | 80% |
| 6min | 90% |
| 8min | 95% |
| 10min | 97% |
| 12min+ | 98→99% 缓爬 |

## 完成动画时长

收到后端完成信号时，动画时长随剩余距离缩放，跳得远就慢一拍：

| 完成时进度 | 动画时长 |
|-----------|---------|
| 95% | 300ms |
| 80% | 300ms |
| 41% | 472ms |
| 17% | 664ms |

## 文案

| 进度区间 | 文案 |
|---------|------|
| 0~30% | 正在处理中… |
| 30~70% | 预计还需几分钟… |
| 70~95% | 即将完成… |
| 95~99%（未超时） | 最后收尾中… |
| 95~99%（已超时） | 仍在处理中，请耐心等待… |
| 100% | 完成 |

## 可配置参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `expectedMs` | 480000 | 预期时长，影响曲线速度 |
| `tickInterval` | 200 | 定时器间隔 ms |
| `maxStep` | 0.5 | 每帧最大步进 % |
| `minStep` | 0.01 | 每帧最小步进 % |
| `overtimeRatio` | 1.5 | 超时倍率，触发兜底文案 |
